このページは、`設問をどう読むか` と `答案の骨子をどう作るか` を整理する解説ページです。与件を読んだあとに、設問要求を外さず解答へつなぐための型を置きます。

<Callout title="このページの役割">
  設問要求を外さず解答へつなぐための型を置くページです。問いの種類、対象、時間軸、制約条件を抜いてから骨子へ進みます。
</Callout>

## 学習のポイント

<Cards>
  <Card title="設問要求">
    `理由`、`課題`、`助言`、`効果` のどれかを先に確定し、答案の方向を固定します。
  </Card>
  <Card title="条件">
    誰に、何について、いつの話か、どんな制約があるかを抜き出してから骨子へ進みます。
  </Card>
  <Card title="骨子">
    `結論`、`根拠`、`施策 / 効果` の順で骨子を作ると、書きながら迷いにくくなります。
  </Card>
</Cards>

## 設問解釈の型

<Steps>
  <Step>
    ### 問いの種類を確定する

    設問は `現状分析`、`原因 / 理由`、`課題抽出`、`助言`、`効果 / 狙い`、`比較` のどれかに分けます。
  </Step>
  <Step>
    ### 主語、対象、時間軸、制約条件を抜く

    同じ助言問題でも、対象顧客、対象工程、短期 / 中長期、制約条件で答え方が変わります。
  </Step>
  <Step>
    ### 骨子は結論から作る

    まず `結論` を短く置き、その後に `与件根拠`、必要なら `具体策`、最後に `効果` をつなぎます。
  </Step>
</Steps>

## 典型例

<Cards>
  <Card title="理由を述べよ">
    施策ではなく `なぜそうなるか` の因果を先に組みます。
  </Card>
  <Card title="助言せよ">
    対象と施策を落とさず `誰に何をどうする` まで入れます。
  </Card>
  <Card title="効果を述べよ">
    売上、関係強化、QCD 改善、資金繰り改善などの成果を明示します。
  </Card>
</Cards>

## 典型的なつまずき

- 設問の動詞を見ずに、毎回同じ型で書く
- 対象顧客や対象工程を落とす
- 根拠のない一般論を骨子に入れる
- 効果を聞かれているのに施策だけで止まる

## 問題を解くときの観点

- 何を答える問題か
- 誰に、何について、いつの話か
- `結論`、`根拠`、`施策 / 効果` のうち何を優先して書くか
