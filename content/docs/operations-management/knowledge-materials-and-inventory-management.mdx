## このページの役割

<Callout title="このページの役割">
    このページは、`切らさない` と `持ちすぎない` を両立するための解説ページです。資材計画と在庫量の考え方を、目的の違いで整理します。
</Callout>

<Callout title="このページの読み方">
    部品が 1 個足りなければラインが止まり、持ちすぎれば倉庫代がかさむ──この「ちょうどよい量」を決める仕組みがこのページのテーマです。まず MRP で「いつ何をいくつ必要か」を展開し、次に EOQ で「1 回に何個発注すると総費用が最小か」を計算し、最後に安全在庫で「需要のブレに備える量」を決めます。
</Callout>

## 学習のポイント

- `MRP` は必要量展開（計画から逆算）
- `EOQ` は発注量の最適化（費用の最小化）
- `安全在庫` は欠品防止（需要変動への備え）
- `ABC分析` は管理の重点付け

## 試験で何が問われるか

- MRP の所要量展開を **計算** できるか
- EOQ の公式を使って **最適発注量** を求められるか
- 安全在庫・発注点の **計算** ができるか
- 定量発注方式と定期発注方式の **違い** を説明できるか

## MRP（資材所要量計画）

MRP は、生産計画（MPS）をもとに部品や資材の必要量をBOM（部品構成表）で展開する仕組みです。

<Callout title="MRPの役割">
    MRP は「何をいつ必要か」を **生産計画から逆算** する仕組みです。生産日時と部品構成から、各部品の発注時期と発注量を決定します。重要なのは **リードタイムオフセット** で、調達に時間がかかるほど早く発注する必要があります。
</Callout>

### MRP の計算手順

```
① 総所要量 = 親品目の計画オーダー × 構成数量
② 正味所要量 = 総所要量 − (手持在庫 + 発注残 − 引当量)
③ ロットまとめ（ロットサイズに合わせる）
④ リードタイムオフセット（LTだけ前倒しして発注時期を決定）
⑤ 下位品目に展開して①〜④を繰り返す
```

### 計算例（2層BOM）

```
【データ】
  製品X 10個を第5週に生産する
  BOM: 製品X 1個 → 部品A ×3個（LT=2週）→ 材料B ×2個（LT=1週）
  部品Aの手持在庫 12個、材料Bの手持在庫 20個

手順1: 部品Aの総所要量
  = 10 × 3 = 30個

手順2: 部品Aの正味所要量
  = 30 − 12 = 18個

手順3: リードタイムオフセット
  部品AのLT=2週 → 第5週−2週 = 第3週に18個発注

手順4: 材料Bの総所要量
  = 18 × 2 = 36個（部品Aの正味所要量 × 構成数量）

手順5: 材料Bの正味所要量
  = 36 − 20 = 16個

手順6: リードタイムオフセット
  材料BのLT=1週 → 第3週−1週 = 第2週に16個発注
```

### 計算例（3層BOM展開）

実務ではBOMが3層以上になることが多いため、段階的な所要量展開と複数部品のリードタイムオフセットが重要です。

```
【データ】
  完成品Y：100個を第10週に完成させる
  BOM：
    Y 1個 → 部品C ×2個（LT=3週） + 部品D ×1個（LT=1週）
    部品C 1個 → 素材E ×3個（LT=2週）
  手持在庫：C=30個、D=0個、E=50個

【展開】

◎ 第10週：完成品Y 100個を生産
  → 部品C必要量 = 100 × 2 = 200個
  → 部品D必要量 = 100 × 1 = 100個

◎ 部品C：手持30個 → 正味所要量 = 200 − 30 = 170個、LT=3週
  → 第10週 − 3週 = 第7週に170個発注

◎ 部品D：手持0個 → 正味所要量 = 100個、LT=1週
  → 第10週 − 1週 = 第9週に100個発注

◎ 素材E：部品C 170個 × 3個 = 510個必要
  → 手持50個 → 正味所要量 = 510 − 50 = 460個、LT=2週
  → 部品C発注日の第7週 − 2週 = 第5週に460個発注

【発注スケジュール】
  第5週：素材E 460個発注
  第7週：部品C 170個発注
  第9週：部品D 100個発注
  第10週：Y 100個完成
```

### ポイント：リードタイムオフセットで発注時期が大きく変わる

- 素材E（LT=2週）は完成まで5週間前に発注
- 部品D（LT=1週）は完成の1週間前に発注
- **発注タイミングを間違うと欠品か過剰在庫のどちらかが起きる**

## EOQ（経済的発注量）

EOQ は、発注費用と在庫保管費用の合計が最小になる 1 回あたりの発注量です。

$$
EOQ = \sqrt{\frac{2DS}{H}}
$$

`D` = 年間需要量、`S` = 1回あたりの発注費用、`H` = 1単位あたりの年間保管費用

### EOQ のポイント

EOQ では **年間発注費用 = 年間保管費用** となります。これが最適条件です。

<Callout title="EOQの本質">
    1回の発注量を増やすと：発注費用は下がるが在庫が増えて保管費用が上がる。逆に発注量を減らすと：保管費用は下がるが発注回数が増えて発注費用が上がる。EOQはこの二つが等しくなる量です。
</Callout>

### 計算例

```
【データ】
  年間需要量 D = 10,000個
  1回あたり発注費用 S = 5,000円
  1個あたり年間保管費用 H = 250円

EOQ = √(2 × 10,000 × 5,000 / 250)
    = √(100,000,000 / 250)
    = √400,000
    ≒ 632個

年間発注回数 = 10,000 ÷ 632 ≒ 15.8回
年間発注費用 = 15.8 × 5,000 ≒ 79,000円
年間保管費用 = (632÷2) × 250 = 316 × 250 ≒ 79,000円
年間総費用 ≒ 158,000円

→ 発注費用 ≒ 保管費用（最適条件を確認）✓
```

### EOQ の感度分析（パラメータ変化の影響）

EOQは需要や費用の変化に対して **意外と鈍感** です。以下の例で確認しましょう。

**シナリオ1：需要が2倍になった場合**

```
【新データ】D' = 20,000個（2倍）、S = 5,000円、H = 250円

EOQ' = √(2 × 20,000 × 5,000 / 250)
     = √800,000,000 / 250
     ≒ 894個

EOQ' / EOQ = 894 / 632 ≒ 1.41（約 √2倍）
```

→ 需要が2倍になってもEOQは√2倍（約1.41倍）の増加にとどまります。

**シナリオ2：保管費用が半分になった場合**

```
【新データ】D = 10,000個、S = 5,000円、H' = 125円（半分）

EOQ' = √(2 × 10,000 × 5,000 / 125)
     = √800,000
     ≒ 894個

EOQ' / EOQ = 894 / 632 ≒ 1.41（約 √2倍）
```

→ 保管費用が半分になってもEOQは√2倍の増加。

**シナリオ3：EOQからずれた場合の費用増加**

```
EOQより20%多い750個で発注した場合：
  発注費用 = 10,000 ÷ 750 × 5,000 = 13.3 × 5,000 ≒ 66,667円
  保管費用 = (750÷2) × 250 = 375 × 250 = 93,750円
  総費用 = 160,417円

EOQで発注した場合：158,000円

費用増加 = (160,417 − 158,000) / 158,000 ≒ 1.5%
```

→ **20%ずれても費用は約1～2%の増加にとどまる**（EOQの鈍感性）。現実の在庫管理では多少のずれは許容範囲です。

## 安全在庫と発注点

### 安全在庫

$$
\text{安全在庫} = k \times \sigma \times \sqrt{L}
$$

`k` = 安全係数、`σ` = 需要の標準偏差、`L` = リードタイム

<Callout title="安全在庫の意味">
    安全在庫は「もしもの時」の備えです。通常の需要予測よりも多く在庫を持つことで、需要が予想より多かったり、納入が遅れたりした場合の欠品を防ぎます。サービス率が高いほど安全在庫は増え、保管コストが上がります。
</Callout>

### サービス率と安全係数

| サービス率 | 安全係数 k | 欠品の頻度 |
| --- | --- | --- |
| 90% | 1.28 | 約36回/年 |
| 95% | 1.65 | 約18回/年 |
| 97% | 1.88 | 約10回/年 |
| 99% | 2.33 | 約5回/年 |
| 99.9% | 3.09 | 約1回/年 |

#### サービス率の感度

```
97%から99%へ：1.88 → 2.33（約1.24倍の増加）
99%から99.9%へ：2.33 → 3.09（約1.33倍の増加）

→ 高いサービス率を求めると、安全在庫が急増する
```

### 発注点

$$
\text{発注点} = \text{1日あたり平均使用量} \times \text{調達LT} + \text{安全在庫}
$$

### 計算例

```
【データ】
  1日あたり平均使用量 50個、調達LT 4日
  需要の標準偏差 σ = 10個/日、サービス率 95%（k=1.65）

安全在庫 = 1.65 × 10 × √4 = 1.65 × 10 × 2 = 33個
発注点 = 50 × 4 + 33 = 233個

→ 在庫が233個に達したら発注する
```

### サービス率の違いによる発注点の変化

```
【同じデータで異なるサービス率を適用】

サービス率 90%（k=1.28）：
  安全在庫 = 1.28 × 10 × 2 = 26個
  発注点 = 200 + 26 = 226個

サービス率 95%（k=1.65）：
  安全在庫 = 1.65 × 10 × 2 = 33個
  発注点 = 200 + 33 = 233個

サービス率 99%（k=2.33）：
  安全在庫 = 2.33 × 10 × 2 = 47個
  発注点 = 200 + 47 = 247個

→ 5%のサービス率向上（90%→95%）で在庫7個増
→ 4%のサービス率向上（95%→99%）で在庫14個増（加速度的に増える）
```

## 発注方式の比較

### 定量発注方式 vs 定期発注方式

| 比較軸 | 定量発注方式 | 定期発注方式 |
| --- | --- | --- |
| 発注のタイミング | 在庫が **発注点** に達した時 | **一定周期**（例：毎週月曜）で発注 |
| 1回の発注量 | 一定量（EOQ） | 都度計算（変動） |
| 適する品目 | B・C品目（管理が簡単） | A品目（精緻な管理） |
| 需要変動への対応 | 発注点を超えたら即発注 | 発注間隔内の需要変動を吸収しにくい |
| 必要な在庫 | 平均的 | 多め（不確実性に対応） |
| 管理の手間 | 常時在庫監視が必要 | 定期確認（例：毎週チェック） |
| 実装の簡易性 | 難（ただし2ビン方式で簡略化可能） | 簡単 |

### 定期発注方式の発注量

$$
\text{発注量} = (\text{発注間隔} + \text{調達LT}) \times \text{予測需要量} + \text{安全在庫} - \text{手持在庫} - \text{発注残}
$$

### 計算例

```
【データ】
  発注間隔 7日、調達LT 3日、予測需要量 40個/日
  安全在庫 50個、手持在庫 120個、発注残 80個

発注量 = (7+3)×40 + 50 − 120 − 80
       = 400 + 50 − 120 − 80
       = 250個
```

### 2ビン方式（簡易版定量発注）

小物部品の管理では、複雑な計算の代わりに **2ビン方式** を使うことがあります。

```
【仕組み】
  ビン1（通常ビン）：日常使用分
  ビン2（予備ビン）：最小在庫分 + 安全在庫

  使用の流れ：
  1. ビン1から取出す
  2. ビン1が空になったら、ビン2から取出す
  3. ビン2を取出したら、同時に発注する（ビン1の補充待ち）
  4. 納品後、ビン1を満杯にして再開

【メリット】
  - 計算が不要で実装が簡単
  - 小物部品に最適
  - 在庫管理システムがなくても運用可能

【デメリット】
  - 需要が大きく変わると対応が難しい
  - ビンのサイズ設定が重要
```

## ABC 分析

ABC 分析は、パレートの法則（上位 20%で全体の 80%を占める）を在庫管理に適用し、重点管理品目を決める手法です。

<Callout title="ABC分析の本質">
    金額で見ると、少数の高価な品目（A品目）が全体費用の大部分を占めています。一方、多数の低価格品目（C品目）は管理負荷は大きいが金額的には小さい。限られた管理リソースを効率的に使うため、高額品を重点管理し、低額品は簡易管理にします。
</Callout>

| ランク | 金額構成比の目安 | 管理方針 | 発注方式 | 管理頻度 |
| --- | --- | --- | --- | --- |
| A品目 | 上位 70-80% | 精緻な管理 | 定期発注方式 | 毎日～毎週 |
| B品目 | 次の 15-20% | 中程度の管理 | 併用 | 週1～月1 |
| C品目 | 残り 5-10% | 簡易な管理 | 定量発注方式 / 2ビン方式 | 月1～随時 |

### ABC分析の計算手順

10品目を例に、金額の累積構成比を計算してA/B/C分類します。

```
【品目データと年間購買額】

品目  単価  年間使用量  購買額
P1    10,000  20      200,000
P2    5,000   50      250,000
P3    100     2,000   200,000
P4    1,000   30      30,000
P5    500     100     50,000
P6    50      1,500   75,000
P7    200     150     30,000
P8    3,000   10      30,000
P9    800     25      20,000
P10   100     100     10,000

合計               895,000

【ステップ1：購買額で降順に並べ直す】

順位  品目  購買額  累積額   構成比  累積構成比
1     P2    250,000  250,000  27.9%  27.9%
2     P1    200,000  450,000  22.4%  50.3%
3     P3    200,000  650,000  22.4%  72.7%
4     P6    75,000   725,000  8.4%   81.1%
5     P5    50,000   775,000  5.6%   86.7%
6     P8    30,000   805,000  3.4%   90.1%
7     P4    30,000   835,000  3.4%   93.5%
8     P7    30,000   865,000  3.4%   96.9%
9     P9    20,000   885,000  2.2%   99.1%
10    P10   10,000   895,000  1.1%   100%

【ステップ2：累積構成比でA/B/C分類】

A品目（～80%）：P2, P1, P3（累積72.7%）
  → 3品目で全体の72.7%をカバー

B品目（80～95%）：P6, P5, P8（累積90.1%）
  → さらに3品目で80%～95%をカバー

C品目（95～100%）：P4, P7, P9, P10（累積100%）
  → 残り4品目で低額品

【管理方針】

A品目（P1, P2, P3）：
  - 毎日在庫確認
  - 定期発注方式（精密に管理）
  - 詳細な記録

B品目（P5, P6, P8）：
  - 週1回確認
  - 定量・定期の併用
  - 標準的な管理

C品目（P4, P7, P9, P10）：
  - 月1回確認
  - 2ビン方式または多めの発注ロット
  - 簡易管理
```

## ロットサイジング手法

MRP で正味所要量が決まった後、「いつ、いくつ発注するか」を決めるのが **ロットサイジング** です。複数期間の需要を一度にまとめて発注するか、毎期必要分だけ発注するかで、在庫と発注費用が大きく変わります。

### 主な手法

```
【1】ロット・フォー・ロット（L4L）
  必要量と必要時期が完全に一致する発注

  例：1週目50個、2週目30個、3週目70個必要
  → 1週目に50個、2週目に30個、3週目に70個を個別発注

  メリット：在庫が最小（ゼロに近い）
  デメリット：発注回数が多い → 発注費用が高い
  用途：高額品目、賞味期限がある品目

【2】固定ロット方式
  一定量（例：100個単位）ずつ発注

  例：1週目50個必要
  → 100個まとめて発注（50個は翌週に繰り越し）

  メリット：発注回数が少ない
  デメリット：在庫が増える
  用途：小物部品、変動が小さい品目

【3】期間まとめ（例：2週間分まとめて）
  複数期間の需要をまとめて発注

  例：1週目50個、2週目30個 → 80個をまとめて第1週に発注

  メリット：L4Lと固定ロットの中間
  デメリット：需要予測の変動に弱い
  用途：中程度の在庫が許容できる品目

【4】最小費用法（Dynamic Programming）
  発注費用 + 在庫保管費用の合計を最小化する発注時期を計算

  複数期間の需要と費用データから最適な発注パターンを探索
  用途：高額品で複数期間先まで計画する場合
```

### 簡易比較例

```
3週間で必要な量：1週目 100個、2週目 80個、3週目 60個
（合計240個）

発注費用：1回3,000円
保管費用：1個1円/週

【L4L：3回に分けて発注】
  発注費用：3,000 × 3 = 9,000円
  保管費用：ほぼ0円
  総費用：約9,000円

【固定ロット（100個単位）】
  発注量：100個→200個→100個（計3回）
  発注費用：3,000 × 3 = 9,000円
  保管費用：
    1週後：20個（100-80=20）保管 → 20円
    2週後：40個（20+100-80=40）保管 → 40円
    3週後：100個（40+100-60=80）保管 → 80円
  総費用：9,000 + 140 ≒ 9,140円

【期間まとめ（2週間分）】
  発注量：180個（100+80）→ 60個（計2回）
  発注費用：3,000 × 2 = 6,000円
  保管費用：
    1週後：80個（180-100=80）保管 → 80円
    2週後：0個（80-80=0）保管 → 0円
    3週後：0個（0-60=0）保管 → 0円
  総費用：6,000 + 80 ≒ 6,080円

→ 高額品ほどロット方式の選択が重要
```

<Callout title="ここまでの整理 ── MRP・EOQ・安全在庫の使い分け">
    MRP は「何をいつ必要か」を計画から逆算する仕組み、EOQ は「1回に何個発注するのがコスト最小か」を費用から計算する仕組み、ロットサイジングは「複数期間の需要をどうまとめて発注するか」を決める仕組み、安全在庫は「需要のブレに備える量」をサービス率から決める仕組みです。それぞれ解く問題が違うので混同しないでください。
</Callout>

## 需要予測 ── 指数平滑法

$$
\text{次期予測値} = \alpha \times \text{前期実績値} + (1 - \alpha) \times \text{前期予測値}
$$

`α` が大きい（1 に近い）→ 直近の変化に敏感。`α` が小さい（0 に近い）→ 過去のトレンド重視。

```
【計算例】
  前期予測値 100、前期実績値 120、α = 0.3
  次期予測値 = 0.3 × 120 + 0.7 × 100 = 36 + 70 = 106
```

## 典型的なつまずき

- MRP、EOQ、安全在庫を同じ在庫論点として一括で覚える → **目的が違う**（計画展開、費用最小化、欠品防止）
- EOQ の公式で分母と分子を逆にする → `2DS` が分子、`H` が分母
- 安全在庫を `余分な在庫` とだけ考えてしまう → サービス率に基づく合理的な量
- ABC分析を数量順の分類だと思ってしまう → **金額** の累積構成比で分類
- 定量発注と定期発注を混同する → 「いつ発注するか」と「いくつ発注するか」で切る
- EOQ で「発注費用 = 保管費用」になることを忘れる → 最適条件の検証に使える

## 問題を解くときの観点

- 問われているのは `必要量` か `発注量` か `欠品防止` か
- 在庫を減らしたいのか、切らしたくないのか
- 重点管理すべき品目は何か
- 計算結果で `発注費用 ≒ 保管費用` になっているか確認

## 確認問題

<Callout title="問1：EOQ の計算">
    年間需要量 8,000個、1回あたり発注費用 4,000円、1個あたり年間保管費用 200円。EOQ を求めよ。

    **解答**：
    EOQ = √(2×8,000×4,000/200) = √(320,000) ≒ 566個

    年間発注費用 = (8,000÷566)×4,000 ≒ 56,537円
    年間保管費用 = (566÷2)×200 = 56,600円
    → 発注費用と保管費用がほぼ等しい（最適条件確認）
</Callout>

<Callout title="問2：発注点の計算">
    1日あたり平均使用量 30個、調達リードタイム 5日、安全在庫 45個。発注点を求めよ。

    **解答**：
    発注点 = 30×5 + 45 = 150 + 45 = 195個

    意味：在庫が195個に達したら発注する。LT中に平均で150個消費されるため、安全在庫45個で欠品防止。
</Callout>

<Callout title="問3：MRP の展開（2層）">
    製品P を 20個生産する。BOM: P 1個 → 部品Q ×2個。部品Qの手持在庫 15個、発注残 5個。正味所要量を求めよ。

    **解答**：
    総所要量 = 20 × 2 = 40個
    正味所要量 = 40 − (15 + 5) = 20個

    解説：手持在庫と発注残を差し引く必要があります。
</Callout>

<Callout title="問4：3層BOM展開とリードタイムオフセット">
    完成品X を第12週に100個完成させたい。
    BOM：X 1個 → 部品A ×2個（LT=3週）
        部品A 1個 → 素材B ×4個（LT=2週）
    手持在庫：A=50個、B=100個

    (1) 部品A の正味所要量と発注時期を求めよ。
    (2) 素材B の正味所要量と発注時期を求めよ。

    **解答**：
    (1) 部品A：
      総所要量 = 100 × 2 = 200個
      正味所要量 = 200 − 50 = 150個
      LT=3週 → 第12週 − 3週 = 第9週に150個発注

    (2) 素材B：
      総所要量 = 150 × 4 = 600個
      正味所要量 = 600 − 100 = 500個
      LT=2週 → 部品A発注日の第9週 − 2週 = 第7週に500個発注

    発注スケジュール：第7週（素材B） → 第9週（部品A） → 第12週（完成品X）
</Callout>

<Callout title="問5：ABC分析の分類">
    5品目の年間購買額：品目1=300,000円、品目2=200,000円、品目3=150,000円、品目4=100,000円、品目5=50,000円。
    各品目を分類し、品目1, 2, 3 の累積構成比を求めよ。

    **解答**：
    合計 = 800,000円

    累積構成比：
    品目1：300,000 ÷ 800,000 = 37.5%（A品目候補）
    品目2：(300,000 + 200,000) ÷ 800,000 = 62.5%（A品目）
    品目3：(300,000 + 200,000 + 150,000) ÷ 800,000 = 81.25%（A品目）

    分類：
    A品目（～80%）：品目1, 品目2（累積62.5%）
         ※品目3は追加で81.25%になるためA品目に含める実装も多い
    B品目（80～95%）：品目4（87.5%）
    C品目（95～100%）：品目5（100%）

    → 2～3品目（全体の62～81%）に管理リソースを集中
</Callout>

<Callout title="問6：定期発注方式の発注量">
    発注間隔 10日、調達LT 4日、予測需要量 25個/日、安全在庫 80個。
    現在の手持在庫 150個、発注残 50個のとき、発注量を求めよ。

    **解答**：
    発注量 = (10 + 4) × 25 + 80 − 150 − 50
           = 14 × 25 + 80 − 150 − 50
           = 350 + 80 − 200
           = 230個

    意味：発注～納品までの14日間分（350個）に安全在庫（80個）を加えて、
    現在の在庫状況（150+50）を差し引くと、追加で230個必要。
</Callout>
