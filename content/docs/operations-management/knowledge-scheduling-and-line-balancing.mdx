## 前提知識

<Callout title="このページを読む前に">
    このページは、プロジェクトや製造ラインで「期限内に、効率よく完成させるための計画」の立て方を学びます。

    前提として以下を理解しておくと読みやすいです：
    - 作業の前後関係（どの作業が終わってから次が始まるか）
    - 機械やステーションの概念（複数の作業を並列に処理）
    - 時間単位の一貫性（分・時間・日を混ぜない）
</Callout>

## まずイメージをつかむ

プロジェクト全体を完成させるには、「どの経路が全体を遅らせるか」を知ることが大事です。
- 建設プロジェクト：鉄骨組み立てが遅れると、その後の内装工事も遅れる（**クリティカルパス**）
- 工場のライン：最も時間のかかる工程のペースに全体が合わせられる（**ボトルネック**）
- 2台の機械で複数の部品を加工：加工順序で総所要時間が大きく変わる（**ジョンソン法**）

このページではこれら 3 つの視点から、スケジューリングと工程管理を整理します。

## 試験で何が問われるか

- クリティカルパスを **求める** か（最長経路の計算）
- 余裕時間（フロート）を **計算** できるか
- 作業を **短縮するとき最適な選択肢** は何か（クラッシング）
- ジョンソン法で最適順序を **導出** できるか
- ラインバランシングで **工程をステーションに割り当てる** ことができるか
- ディスパッチングルールの **特性を使い分ける** ことができるか

## PERT/CPM ── 最短工期とクリティカルパス

PERT（Program Evaluation and Review Technique）/CPM（Critical Path Method）は、作業の前後関係をネットワーク図で表し、最短工期と余裕のない経路（クリティカルパス）を求める手法です。

### 基本用語

| 用語 | 意味 |
| --- | --- |
| 最早開始時刻（ES） | その作業を最も早く始められる時刻 |
| 最早完了時刻（EF） | ES + 作業時間 |
| 最遅完了時刻（LF） | その作業を遅くとも完了すべき時刻 |
| 最遅開始時刻（LS） | LF − 作業時間 |
| 総余裕時間（TF） | LS − ES（= LF − EF） |
| クリティカルパス | TF = 0 の作業を結ぶ最長経路 |

### 計算手順

```
【フォワードパス（前進計算）】── 最早時刻を求める
① 開始ノードの ES = 0
② 各作業の EF = ES + 作業時間
③ 合流ノードの ES = 流入する全作業の EF の最大値

【バックワードパス（後退計算）】── 最遅時刻を求める
④ 最終ノードの LF = 最短工期（フォワードパスの結果）
⑤ 各作業の LS = LF − 作業時間
⑥ 分岐ノードの LF = 流出する全作業の LS の最小値

【余裕時間】
⑦ TF = LS − ES（= LF − EF）
⑧ TF = 0 の作業がクリティカルパス上の作業
```

### 計算例

```
【ネットワーク】
  作業A(3日) → 作業C(4日) → 作業E(2日)
  作業B(5日) → 作業D(3日) → 作業E(2日)
  作業A(3日) → 作業D(3日)

【フォワードパス】
  A: ES=0, EF=3
  B: ES=0, EF=5
  C: ES=3, EF=7
  D: ES=max(3,5)=5, EF=8
  E: ES=max(7,8)=8, EF=10

→ 最短工期 = 10日

【バックワードパス】
  E: LF=10, LS=8
  D: LF=8, LS=5
  C: LF=8, LS=4
  A: LF=min(4,5)=4, LS=1
  B: LF=5, LS=0

【余裕時間（TF = LS − ES）】
  A: 1−0=1日  B: 0−0=0日  C: 4−3=1日  D: 5−5=0日  E: 8−8=0日

→ クリティカルパス: B → D → E（TF=0の経路）
→ この経路上の作業が遅れると全体工期が延びる
```

### 3点見積法（PERT）

作業時間が不確実な場合、楽観値（a）、最頻値（m）、悲観値（b）の3つの推定値から期待時間と分散を算出します。

**期待時間の公式：**

$$
t_e = \frac{a + 4m + b}{6}
$$

**分散の公式：**

$$
\sigma^2 = \left( \frac{b - a}{6} \right)^2
$$

**計算例**

ある作業について、以下の見積もりがあります：
- 楽観値（a）：3日
- 最頻値（m）：5日
- 悲観値（b）：13日

期待時間：

$$
t_e = \frac{3 + 4(5) + 13}{6} = \frac{3 + 20 + 13}{6} = \frac{36}{6} = 6日
$$

分散：

$$
\sigma^2 = \left( \frac{13 - 3}{6} \right)^2 = \left( \frac{10}{6} \right)^2 = 2.78
$$

標準偏差：σ = √2.78 ≈ 1.67日

**意味**：この作業は平均6日で完了すると予想されますが、±1.67日の変動がある（信頼度約68%）ことを意味します。

複数の作業からなるプロジェクトの場合、各作業の期待時間を合計してプロジェクト全体の期待工期を求め、クリティカルパス上の作業の分散を合計してプロジェクト全体の分散を算出できます。

### クラッシング（工期短縮）

計画工期を前倒しする必要が生じたとき、クリティカルパス上のどの作業を短縮すべきか、という意思決定の問題です。

**基本原則**

- **短縮できるのはクリティカルパス上の作業のみ**：クリティカルパス以外を短縮しても全体工期は変わらない
- **コスト最小短縮の手順**：複数の短縮候補がある場合、「短縮コスト/日」が最小の作業から短縮を検討する

**計算例**

以下のプロジェクト（クリティカルパス：B→D→E、工期10日）について、1日短縮する場合を考えます。

```
【各作業の短縮可能性】
  作業 | 通常時間 | 短縮可能時間 | 追加コスト | 短縮コスト/日
  ─────┼─────────┼──────────┼─────────┼────────────
  B   |   5日    |   2日     | 100万円  | 50万円/日
  D   |   3日    |   1日     | 60万円   | 60万円/日
  E   |   2日    |   1日     | 40万円   | 40万円/日 ← 最小
```

1日短縮する場合は、**作業E（短縮コスト40万円/日）を1日短縮**するのが最適です。

もし2日短縮が必要なら、まずEを1日短縮（合計40万円）、その後Bを1日短縮（合計40+50=90万円）となります。

## ジョンソン法 ── 2機械フローショップの最適順序

ジョンソン法は、2 台の機械で n 個のジョブを処理する場合に、総所要時間を最小化する順序を求める方法です。

### アルゴリズム

```
① 全ジョブの機械1と機械2の処理時間をリストアップ
② 全体の最小値を探す
③ その値が機械1にあれば → そのジョブを先頭から順に入れる
④ その値が機械2にあれば → そのジョブを末尾から順に入れる
⑤ 決定したジョブを除外して②に戻る
⑥ 全ジョブの順序が決まるまで繰り返す
```

### 計算例1：基本的なジョンソン法

```
【データ】4つのジョブ
  ジョブ | 機械1 | 機械2
  ───────┼──────┼──────
  J1     |   3   |   6
  J2     |   8   |   2
  J3     |   5   |   7
  J4     |   4   |   3

手順1: 全体の最小値 → J2の機械2=2
  → 機械2なので末尾に配置: [_, _, _, J2]

手順2: 残り{J1,J3,J4}の最小値 → J1の機械1=3
  → 機械1なので先頭に配置: [J1, _, _, J2]

手順3: 残り{J3,J4}の最小値 → J4の機械2=3
  → 機械2なので末尾から配置: [J1, _, J4, J2]

手順4: 残り{J3}
  → [J1, J3, J4, J2]

最適順序: J1 → J3 → J4 → J2
総所要時間: 機械1で J1(3) → J3(5) → J4(4) → J2(8) = 20分
```

### 計算例2：タイがある場合の処理

ジョンソン法で同じ値が複数ある場合、**どの順序で処理しても結果は同じ**です。任意に選んで進めても構いません。

```
【データ】タイがある場合
  ジョブ | 機械1 | 機械2
  ───────┼──────┼──────
  J1     |   4   |   3
  J2     |   3   |   3 ← タイ（機械1と機械2が同じ）
  J3     |   2   |   5

手順1: 最小値 = J3の機械1=2
  → 機械1なので先頭: [J3, _, _]

手順2: 残り{J1,J2}の最小値 = J2の機械1=3 と機械2=3（タイ）
  → どちらを採用しても OK。ここでは機械1を採用 → 先頭: [J3, J2, _]

手順3: 残り{J1}
  → [J3, J2, J1]

最適順序: J3 → J2 → J1
```

### 3機械以上の場合（簡易法）

ジョンソン法は厳密には2機械用です。3機械以上の場合は、**2つの架空機械に変換する簡易法**を用いることがあります。

```
【簡易法の考え方】
機械が M1, M2, M3 の場合：
- 架空機械α = M1の処理時間 + M2の処理時間
- 架空機械β = M2の処理時間 + M3の処理時間

このα, βに対してジョンソン法を適用する
```

例：
```
  ジョブ | M1 | M2 | M3
  ───────┼────┼────┼────
  J1     | 3  | 2  | 4
  J2     | 5  | 1  | 3
  J3     | 2  | 4  | 2

架空機械化：
  ジョブ | α(M1+M2) | β(M2+M3)
  ───────┼──────────┼─────────
  J1     |    5     |    6
  J2     |    6     |    4  ← 最小
  J3     |    6     |    6

ジョンソン法適用 → J2 → J1 → J3 など
（ただし3機械以上は厳密解ではなく参考値）
```

## ラインバランシング

ラインバランシングは、各工程への作業配分を最適化し、ライン全体の効率を高める手法です。

### タクトタイム

$$
\text{タクトタイム} = \frac{\text{稼働時間}}{\text{必要生産数}}
$$

タクトタイムは「市場が要求する1個あたりの生産ペース」です。各工程の作業時間はこのタクトタイム以内に収める必要があります。

### サイクルタイム

実際のラインにおけるサイクルタイム（CT）は、最も時間のかかる工程（ボトルネック工程）の作業時間です。

$$
CT = \max(\text{各工程の作業時間})
$$

### ライン編成効率

$$
\text{ライン編成効率} = \frac{\text{各工程の作業時間の合計}}{\text{工程数} \times CT} \times 100\%
$$

### バランスロス率

$$
\text{バランスロス率} = 100\% - \text{ライン編成効率}
$$

### 計算例1：効率を評価する

```
【データ】
  稼働時間 480分/日、必要生産数 120個/日
  工程A: 3分、工程B: 4分、工程C: 2分、工程D: 3分

手順1: タクトタイム
  = 480 ÷ 120 = 4分/個

手順2: サイクルタイム（ボトルネック）
  CT = max(3, 4, 2, 3) = 4分（工程B）

手順3: ライン編成効率
  = (3+4+2+3) ÷ (4×4) × 100
  = 12 ÷ 16 × 100
  = 75%

手順4: バランスロス率
  = 100 − 75 = 25%

→ 25%の時間がムダ待ちになっている
→ タクトタイム=CT=4分なので、生産ペースは市場要求ぴったり
→ 工程Bがボトルネック。改善するなら工程Bの作業時間短縮
```

### 計算例2：工程をステーションに割り当てる

実際の生産ラインでは、複数の作業を1つのステーション（工程グループ）に割り当てます。先行関係を守りながら、タクトタイム以内に各ステーションの作業を完了させる必要があります。

```
【データ】
  タクトタイム: 60秒
  6つの作業（先行関係あり）
  作業1(20秒) → 作業2(35秒)
             → 作業3(25秒)
  作業2 → 作業4(40秒)
  作業3 → 作業4
  作業4 → 作業5(15秒)
  作業5 → 作業6(30秒)

【最小ステーション数の理論値】
  合計作業時間 = 20+35+25+40+15+30 = 165秒
  最小ステーション数 = ⌈165÷60⌉ = ⌈2.75⌉ = 3ステーション

【ステーション割当（タクトタイム≦60秒を守る）】

  ステーション1（同時並列）：
    作業1(20秒) + 作業2(35秒) = 55秒 OK

  ステーション2：
    作業3(25秒) + 作業4(40秒) = 65秒 → オーバー
    → 作業3(25秒)のみ配置 = 25秒

  ステーション3：
    作業4(40秒) + 作業5(15秒) = 55秒 OK

  ステーション4：
    作業6(30秒) = 30秒

  実配置: 4ステーション必要（理論値3より1多い）

【改善案】
  - 作業4をM段階に分割して複数ステーション割当可能か検討
  - 作業2や作業3の作業時間を短縮できないか検討
```

### 最小工程数の理論値

$$
\text{最小工程数} = \left\lceil \frac{\text{作業時間の合計}}{CT} \right\rceil
$$

（切り上げ）

この値は理論的下限です。実際には先行関係の制約があるため、この値より多くのステーションが必要になることがほとんどです。

## ディスパッチングルール

複数のジョブが機械を待っているとき、どの順序で処理するかを決めるルール。目的によって最適なルールは異なります。

| ルール | 基準 | 最適化対象 | 特徴 |
| --- | --- | --- | --- |
| SPT（最短処理時間） | 処理時間が短い順 | 平均滞留時間 | 平均滞留時間が最小になる |
| EDD（最早納期） | 納期が早い順 | 納期遅れ | 最大納期遅れを最小化 |
| FCFS（先着順） | 到着順 | 公平性 | 公平だが効率は低い |

### ディスパッチングルール比較の計算例

4つのジョブがあり、処理機械は1台です。現在時刻は0とします。

```
【ジョブデータ】
  ジョブ | 処理時間 | 納期
  ───────┼─────────┼─────
  J1     |   5分    | 12分
  J2     |   3分    | 6分
  J3     |   8分    | 15分
  J4     |   4分    | 9分
```

#### SPT（最短処理時間順）

処理時間の短い順：J2(3) → J4(4) → J1(5) → J3(8)

```
  ジョブ | 開始時刻 | 完了時刻 | 滞留時間 | 納期遅れ
  ───────┼─────────┼─────────┼─────────┼────────
  J2     |    0    |    3     |    3    |   0
  J4     |    3    |    7     |    7    |   0
  J1     |    7    |   12     |   12    |   0
  J3     |   12    |   20     |   20    |   5

  平均滞留時間 = (3+7+12+20) ÷ 4 = 10.5分
  最大納期遅れ = 5分
```

#### EDD（最早納期順）

納期の早い順：J2(6) → J4(9) → J1(12) → J3(15)

```
  ジョブ | 開始時刻 | 完了時刻 | 滞留時間 | 納期遅れ
  ───────┼─────────┼─────────┼─────────┼────────
  J2     |    0    |    3     |    3    |   0
  J4     |    3    |    7     |    7    |   0
  J1     |    7    |   12     |   12    |   0
  J3     |   12    |   20     |   20    |   5

  平均滞留時間 = (3+7+12+20) ÷ 4 = 10.5分
  最大納期遅れ = 5分
```

（この例では J1, J4, J3 の順序が重なり、結果が一致しました）

#### FCFS（到着順）

到着順（問題で指定される順）：J1 → J2 → J3 → J4

```
  ジョブ | 開始時刻 | 完了時刻 | 滞留時間 | 納期遅れ
  ───────┼─────────┼─────────┼─────────┼────────
  J1     |    0    |    5     |    5    |   0
  J2     |    5    |    8     |    8    |   2
  J3     |    8    |   16     |   16    |   1
  J4     |   16    |   20     |   20    |  11

  平均滞留時間 = (5+8+16+20) ÷ 4 = 12.25分
  最大納期遅れ = 11分
```

**結論**

- **平均滞留時間を最小化** したい → **SPT** を使用
- **納期遅れを最小化** したい → **EDD** を使用
- 上例では SPT と EDD が同じ効果でしたが、データによって異なります

## ガントチャート ── 計画と実績を可視化する

ガントチャートは、時間軸の上に作業や製品の進捗を棒グラフで表します。2つの見方があります。

### 機械別ガントチャート（負荷図）

各機械がどの時刻にどのジョブを処理しているかを表します。機械の稼働状況とアイドル時間を可視化できます。

```
【データ】
  ジョブの処理順序：J1 → J3 → J4 → J2
  ジョブ | 機械1での時間 | 機械2での時間
  ───────┼──────────────┼──────────────
  J1     |      3分      |      6分
  J3     |      5分      |      7分
  J4     |      4分      |      3分
  J2     |      8分      |      2分

【機械別ガントチャート】
  機械1：|─J1─|────J3────|──J4─|───────J2────────|
  機械2：|────────J1────────|─────J3─────|─J4─|J2|
          0     3          8     13        20   23 25

  機械1の総稼働時間: 20分
  機械2の総稼働時間: 18分
  機械2のアイドル: 2分（ジョブ完了待ち）
```

**読み方**：機械2が早く終了すれば、その後は機械1から次のジョブを待つアイドル時間が生じます。

### 製品別ガントチャート（スケジュール図）

各製品がどのステーションで、いつ処理されているかを時間順に追跡します。

```
【ラインバランシングの例】
  製品A: |──ステーション1(55秒)──|─ステーション2(25秒)─|──ステーション3(55秒)──|─ステーション4(30秒)─|

  製品B:                      |──ステーション1(55秒)──|─ステーション2(25秒)─|──ステーション3(55秒)──|

  （次のタクトタイム=60秒ごとに製品が流れる）
```

**読み方**：各タクトタイムで1個の製品がライン全体を通過していきます。最後のステーションが完了するまでのリード時間が納期となります。

---

## 公式の前に「なぜ」── 重要な考え方

### PERT/CPMが必要な理由

大規模プロジェクトでは、すべての作業をリアルタイムで把握することは不可能です。だから **「全体を遅らせている経路（クリティカルパス）」を事前に特定** して、そこに注力する。これが PERT/CPM の本質です。

### ジョンソン法が有効な理由

2台の機械がいるとき、単純に「短いジョブから」では最適順序が得られません。なぜなら、**機械2の待機時間を最小化する** という目的を考慮する必要があるからです。だからこそ「機械1なら先頭、機械2なら末尾」というルールが生まれました。

### ラインバランシングの本質

ライン全体の速度は、**最も遅い工程** で決まります（ボトルネック）。どんなに他の工程を高速化しても、ボトルネック工程が変わらなければ全体は改善されません。

---

### よくある計算ミス

| ミス | 原因 | 対策 |
| --- | --- | --- |
| クリティカルパスが複数ある場合に1本だけ報告 | 「最長経路は1本」という先入観 | 全作業の余裕時間を計算して TF=0 をすべて拾う |
| 3点見積法で「期待値」と「標準偏差」を混ぜる | 分散の単位と時間の単位を意識していない | σ² = ((b-a)/6)² で必ず分散から計算 |
| ジョンソン法で「最小値を機械1なら先頭」の判断を誤る | 「最小値の場所」を見落とす | 表を作り、どの機械のどのジョブが最小かを明確に |
| ラインバランシングでタクトタイムと CT を混同 | 「理想」と「現実」を区別していない | タクトタイム = 市場要求、CT = 現在のボトルネック |
| ステーション割当で先行関係を無視 | タクトタイム内に詰め込もうとする | 先行関係を満たす順序を最優先に、その後工程をグループ化 |
| ディスパッチングルールで「絶対に納期遅れがない」と思う | SPT や EDD は統計的最適であって、個別ジョブを保証しない | 「平均的に」「最大値が」最小化されるという条件付き |

## 問題を解くときのチェックリスト

| 項目 | 確認ポイント |
| --- | --- |
| **問われている事項** | 全体日程？ / 最適順序？ / 工程配分？ / 効率評価？ |
| **ネットワーク構造** | 並列作業はあるか？ / 合流点の扱いは正しいか？ |
| **クリティカルパス** | TF=0 の作業を複数見つけたか？ |
| **時間単位** | 分、時間、日が混在していないか？ |
| **タクトタイム vs CT** | タクトタイム < CT → 改善が必要 / タクトタイム ≧ CT → 生産ペース OK |
| **先行関係** | ジョブの前後関係を正しく反映しているか？ |
| **計算検算** | 最早時刻の合計 = 最遅時刻の合計？ |

## 確認問題

<Callout title="問1：クリティカルパス（複数経路）">
    作業A(2日)→C(5日)、作業B(4日)→D(3日)、作業C→E(1日)、作業D→E(1日)の場合、最短工期とクリティカルパスを求めよ。

    **解答**：

    A→C→E: 2+5+1=8日

    B→D→E: 4+3+1=8日

    どちらも8日で、両方がクリティカルパス。最短工期は8日。
</Callout>

<Callout title="問2：3点見積法">
    ある作業の見積もり：楽観値=4日、最頻値=6日、悲観値=14日。期待時間と標準偏差を求めよ。

    **解答**：

    期待時間 = (4 + 4×6 + 14) ÷ 6 = (4 + 24 + 14) ÷ 6 = 42 ÷ 6 = 7日

    分散 = ((14-4)/6)² = (10/6)² = 100/36 ≈ 2.78

    標準偏差 = √2.78 ≈ 1.67日
</Callout>

<Callout title="問3：ライン編成効率">
    5工程のラインで、各工程の作業時間が 6分、8分、5分、7分、4分。ライン編成効率を求めよ。

    **解答**：

    CT = max(6,8,5,7,4) = 8分

    合計 = 6+8+5+7+4 = 30分

    編成効率 = 30÷(5×8)×100 = 30÷40×100 = 75%
</Callout>

<Callout title="問4：ジョンソン法">
    3つのジョブ（J1: 機械1=2,機械2=5、J2: 機械1=6,機械2=3、J3: 機械1=4,機械2=4）の最適順序をジョンソン法で求めよ。

    **解答**：

    最小値 = J1の機械1(2) → 先頭: [J1,_,_]

    次の最小値 = J2の機械2(3) → 末尾: [J1,_,J2]

    残り J3 → [J1,J3,J2]

    最適順序: J1 → J3 → J2
</Callout>

<Callout title="問5：クラッシングの意思決定">
    以下のクリティカルパスがあります。1日短縮する場合、どの作業を短縮すべきか。

    作業 B(5日, 短縮可能1日, 追加コスト50万円)

    作業 D(3日, 短縮可能1日, 追加コスト60万円)

    作業 E(2日, 短縮可能1日, 追加コスト40万円)

    **解答**：短縮コスト/日が最小の作業 E（40万円/日）を1日短縮する。
</Callout>

<Callout title="問6：ディスパッチングルール（SPT vs EDD）">
    4つのジョブ（J1: 処理3分, 納期8分; J2: 処理5分, 納期10分; J3: 処理2分, 納期5分; J4: 処理4分, 納期12分）について、SPT 順と EDD 順で最大納期遅れを比較せよ。

    **解答**：

    SPT 順（2→4→3→5）: J3 は完了時刻7分、納期5分 → 遅れ2分

    EDD 順（3→1→2→4）: J3 は完了時刻2分、納期5分 → 遅れなし

    EDD の方が最大遅れが小さい（0分）。
</Callout>
