## このページの役割

<Callout title="このページの役割">
    このページは、`設備を止めない` と `生産性を上げる` をまとめて整理する解説ページです。故障を減らす話と、
    効率を上げる話を同じ改善文脈で扱います。
</Callout>

<Callout title="このページの読み方">
  設備が止まれば生産が止まり、動いていても速度低下や不良品が出れば効率は下がります。OEE（設備総合効率）はこれら 3 つのロスをまとめて測る指標です。計算問題として頻出するため、公式だけでなく「どの数値がどのロスに対応するか」を理解しましょう。
</Callout>

## 学習のポイント

- `TPM` は全員参加の設備保全
- `OEE` は設備総合効率（3つの率の積）
- `MTBF` と `MTTR` は意味を逆にしない
- `直列` と `並列` のシステム信頼度は計算必須

## 試験で何が問われるか

- OEE を **計算** できるか
- MTBF、MTTR、可用率を **計算** できるか
- 直列・並列システムの信頼度を **計算** できるか
- TPM と各種保全方式の違いを説明できるか

## 設備保全の体系

| 保全方式 | 内容 | タイミング |
| --- | --- | --- |
| 事後保全（BM） | 故障してから修理 | 故障発生後 |
| 予防保全（PM） | 故障する前に整備 | 定期的 or 状態監視 |
| 改良保全（CM） | 故障しにくく設計改良 | 計画的に実施 |
| 保全予防（MP） | 保全しやすい設備を設計 | 設備導入時 |

### TPM（Total Productive Maintenance）

TPM は、全員参加で設備の効率を極限まで高める活動です。設備部門だけでなく、オペレーターが行う `自主保全` が特徴です。

| TPM の柱 | 内容 |
| --- | --- |
| 自主保全 | オペレーターが日常点検・給油・増し締めを行う |
| 計画保全 | 保全部門が計画的に設備を整備する |
| 個別改善 | 6大ロスを分析して設備効率を向上させる |

## OEE（設備総合効率）

$$
OEE = \text{時間稼働率} \times \text{性能稼働率} \times \text{良品率}
$$

### 各率の定義

$$
\text{時間稼働率} = \frac{\text{負荷時間} - \text{停止時間}}{\text{負荷時間}}
$$

$$
\text{性能稼働率} = \frac{\text{基準サイクルタイム} \times \text{加工数量}}{\text{稼働時間}}
$$

$$
\text{良品率} = \frac{\text{加工数量} - \text{不良数量}}{\text{加工数量}}
$$

### 6大ロス

| ロス分類 | 対応する率 | ロスの内容 |
| --- | --- | --- |
| 故障ロス | 時間稼働率 | 突発故障による停止 |
| 段取り・調整ロス | 時間稼働率 | 品種切替による停止 |
| 空転・チョコ停ロス | 性能稼働率 | 一時的な停止 |
| 速度低下ロス | 性能稼働率 | 設計速度を下回る |
| 不良・手直しロス | 良品率 | 不良品の発生 |
| 立上りロス | 良品率 | 安定するまでの不良 |

### 計算例

```
【データ】
  負荷時間 480分、停止時間 60分（故障30分+段取り30分）
  基準サイクルタイム 0.5分/個、加工数量 760個
  不良数量 20個

手順1: 稼働時間
  = 480 − 60 = 420分

手順2: 時間稼働率
  = (480−60) ÷ 480 = 420 ÷ 480 = 0.875（87.5%）

手順3: 性能稼働率
  = (0.5 × 760) ÷ 420 = 380 ÷ 420 ≒ 0.905（90.5%）

手順4: 良品率
  = (760−20) ÷ 760 = 740 ÷ 760 ≒ 0.974（97.4%）

手順5: OEE
  = 0.875 × 0.905 × 0.974
  ≒ 0.771（77.1%）

→ 世界クラスの目標は OEE 85%以上
→ この例では時間稼働率の改善（故障・段取り削減）が最も効果的
```

## MTBF・MTTR・可用率

### 公式

$$
MTBF = \frac{\text{総稼働時間}}{\text{故障回数}}
$$

$$
MTTR = \frac{\text{総修復時間}}{\text{故障回数}}
$$

$$
\text{可用率（アベイラビリティ）} = \frac{MTBF}{MTBF + MTTR}
$$

| 指標 | 意味 | 大きいほど |
| --- | --- | --- |
| MTBF | 平均故障間隔（壊れにくさ） | 信頼性が高い |
| MTTR | 平均修復時間（直しやすさ） | 保全性が **低い** |
| 可用率 | 使える時間の割合 | 稼働率が高い |

### 計算例

```
【データ】
  総運用時間 1,000時間、故障 4回、総修復時間 40時間

手順1: 総稼働時間
  = 1,000 − 40 = 960時間

手順2: MTBF
  = 960 ÷ 4 = 240時間

手順3: MTTR
  = 40 ÷ 4 = 10時間

手順4: 可用率
  = 240 ÷ (240+10) = 240 ÷ 250 = 0.96（96%）
```

## システム信頼度 ── 直列と並列

### 直列システム

全ての要素が動いて初めてシステムが動く場合。

$$
R_{\text{直列}} = R_1 \times R_2 \times \cdots \times R_n
$$

### 並列システム

1つでも動いていればシステムが動く場合。

$$
R_{\text{並列}} = 1 - (1-R_1)(1-R_2) \cdots (1-R_n)
$$

### 計算例

```
【データ】
  装置A: 信頼度 0.9、装置B: 信頼度 0.8

【直列の場合（AとBの両方が必要）】
  R = 0.9 × 0.8 = 0.72

【並列の場合（AかBのどちらかが動けばOK）】
  R = 1 − (1−0.9)(1−0.8) = 1 − 0.1×0.2 = 1 − 0.02 = 0.98

【直列+並列の複合例】
  AとBが並列 → その結果とCが直列（C: 信頼度 0.95）
  並列部分: 0.98
  全体: 0.98 × 0.95 = 0.931
```

---

## 典型的なつまずき

- MTBF と MTTR の意味を逆に覚える → MTBF は **故障間隔**（大きい=壊れにくい）、MTTR は **修復時間**（小さい=直しやすい）
- OEE を単なる稼働率と同じだと思ってしまう → OEE は **3つの率の積**。稼働率が高くても速度低下や不良で下がる
- 設備保全と生産性向上を別々の世界だと考えてしまう → 設備のロス削減が **直接** 生産性向上につながる
- 並列信頼度を「足し算」する → `1−(1−R1)(1−R2)` が正しい
- 可用率の分母を間違える → 分母は `MTBF + MTTR`（全時間のうち稼働割合）

## 問題を解くときの観点

- 問われているのは `止まりにくさ`（MTBF）か `直しやすさ`（MTTR）か `総合効率`（OEE）か
- ロスの原因は停止か、速度か、不良か
- システム構成は直列か、並列か、複合か
- 改善の狙いは設備安定か、作業効率か

## 確認問題

<Callout title="問1：OEE の計算">
    負荷時間 500分、停止時間 50分、基準サイクルタイム 0.4分/個、加工数量 1,000個、不良数量 50個。OEE を求めよ。

    **解答**：時間稼働率 = (500−50)÷500 = 0.90。性能稼働率 = (0.4×1000)÷450 = 400÷450 ≒ 0.889。良品率 = (1000−50)÷1000 = 0.95。OEE = 0.90×0.889×0.95 ≒ 0.760（76.0%）。
</Callout>

<Callout title="問2：可用率の計算">
    MTBF = 200時間、MTTR = 8時間。可用率を求めよ。

    **解答**：可用率 = 200÷(200+8) = 200÷208 ≒ 0.962（96.2%）。
</Callout>

<Callout title="問3：システム信頼度">
    装置A（信頼度0.95）と装置B（信頼度0.90）が並列に接続され、その結果と装置C（信頼度0.85）が直列に接続されている。システム全体の信頼度を求めよ。

    **解答**：並列部分 = 1−(1−0.95)(1−0.90) = 1−0.05×0.10 = 1−0.005 = 0.995。全体 = 0.995×0.85 ≒ 0.846。
</Callout>
