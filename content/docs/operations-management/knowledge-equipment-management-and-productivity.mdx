## このページの役割

<Callout title="このページの役割">
    このページは、`設備を止めない` と `生産性を上げる` をまとめて整理する解説ページです。故障を減らす話と、
    効率を上げる話を同じ改善文脈で扱います。
</Callout>

<Callout title="このページの読み方">
  設備が止まれば生産が止まり、動いていても速度低下や不良品が出れば効率は下がります。OEE（設備総合効率）はこれら 3 つのロスをまとめて測る指標です。計算問題として頻出するため、公式だけでなく「どの数値がどのロスに対応するか」を理解しましょう。
</Callout>

## 学習のポイント

- `TPM` は全員参加の設備保全
- `OEE` は設備総合効率（3つの率の積）
- `MTBF` と `MTTR` は意味を逆にしない
- `直列` と `並列` のシステム信頼度は計算必須

## 試験で何が問われるか

- OEE を **計算** できるか
- MTBF、MTTR、可用率を **計算** できるか
- 直列・並列システムの信頼度を **計算** できるか
- TPM と各種保全方式の違いを説明できるか

## 設備保全の体系と選択

<Callout title="保全方式の背景">
    設備は **必ず劣化** します。その劣化にどう対応するか（いつ、だれが、どのように整備するか）で、故障率・保全コスト・生産性が大きく変わります。
</Callout>

### 4 つの保全方式の比較

| 保全方式 | 実施タイミング | コスト | 適用場面 | リスク |
| --- | --- | --- | --- | --- |
| **事後保全（BM）** | 故障してから修理 | 低（待つ） | 非重要設備、消耗部品 | **大**（突然止まる、他にも巻き込む） |
| **予防保全（PM）** | 定期的に整備、または状態監視で整備 | 中（計画的） | 汎用旋盤、ポンプなど | 小（計画的な停止） |
| **改良保全（CM）** | 故障パターンに基づいて設計改良 | 中～大（改造） | 頻繁な故障部品、ボトルネック設備 | 小（根本改善） |
| **保全予防（MP）** | 新設備導入時に、保全のしやすさを設計に組み込む | 最初は高（設計工夫） | 新規投資の設備 | 最小（最初から堅牢） |

#### 各方式の詳細

**事後保全（Breakdown Maintenance）**
- 「故障してから直す」最もシンプルな方式
- ただし突発的な停止は他工程に波及し、納期遅延やコスト損失が大きい
- 小規模工場や非重要な補助設備向け

**予防保全（Preventive Maintenance）**
- 定期的（例：月1回、5,000時間ごと）に点検・部品交換を実施
- 機械の劣化を先読みし、計画的に停止。生産スケジュール調整が容易
- ほとんどの製造現場の主流

**改良保全（Corrective Maintenance）**
- 同じ部品が繰り返し故障する場合、その原因を改造で排除
- 例：ベルト切れが多い → テンショナー改良、または材質変更
- 予防保全の履歴から「どこが何度も壊れるか」を分析して実施

**保全予防（Maintainability Prevention）**
- 新設備購入・導入時点で「保全しやすさ」を設計に組み込む
- 例：部品交換箇所は手の届く位置に、ボルト規格を統一、油脂給油口を明示
- 後付け改造より はるかに安い。初期投資で将来の保全コスト・時間を削減

### TPM（Total Productive Maintenance）

TPM は、全員参加で設備の効率を極限まで高める活動です。設備部門だけでなく、オペレーターが行う `自主保全` が特徴です。

#### TPM の 8 本柱

| TPM の柱 | 内容 | 推進者 |
| --- | --- | --- |
| 自主保全 | オペレーターが日常点検・給油・増し締めを行い、設備の簡易保全を実施 | オペレーター |
| 計画保全 | 設備の劣化予測に基づいて定期的に点検・交換・修理を計画実施 | 保全技術者 |
| 個別改善 | 6大ロスを分析し、各部門がボトルネック改善を実施 | 全部門 |
| 教育訓練 | 運転・保全・改善の知識スキルを体系的に教育 | 全員 |
| 初期管理 | 新設備導入時に保全のしやすさを設計に組み込む | 企画・設計 |
| 品質保全 | 設備の劣化が品質に及ぼす影響を把握し、品質トラブルを防止 | 品質・生産 |
| 管理間接 | 事務・営業部門を含む全部門の生産性向上を図る | 全部門 |
| 安全衛生環境 | 設備の安全性、作業環境、環境への配慮を向上 | 全員 |

#### 自主保全の 4 ステップ

自主保全は単なる点検ではなく、段階的に設備への理解を深めるプロセスです。

1. **ステップ1：初期清掃** - 設備全体を徹底清掃し、清掃を通じて設備を知る
2. **ステップ2：発生源対策** - 汚れ・振動・異音などの原因を改善し、清掃しやすく
3. **ステップ3：点検基準作り** - 日常点検の項目・方法・基準を文書化
4. **ステップ4：自主管理** - 毎日の点検・給油・調整・記録を実施し、異常を早期発見

## OEE（設備総合効率）

$$
OEE = \text{時間稼働率} \times \text{性能稼働率} \times \text{良品率}
$$

### 各率の定義

$$
\text{時間稼働率} = \frac{\text{負荷時間} - \text{停止時間}}{\text{負荷時間}}
$$

$$
\text{性能稼働率} = \frac{\text{基準サイクルタイム} \times \text{加工数量}}{\text{稼働時間}}
$$

$$
\text{良品率} = \frac{\text{加工数量} - \text{不良数量}}{\text{加工数量}}
$$

### 6大ロスと OEE のマッピング

| ロス分類 | 発生段階 | 対応する率 | ロスの内容 | 影響度 |
| --- | --- | --- | --- | --- |
| **故障ロス** | 設備が止まる | 時間稼働率 | 突発故障による停止時間 | **最大**（稼働率×他の率） |
| **段取り・調整ロス** | 設備が止まる | 時間稼働率 | 品種切替、セットアップ時間 | **大**（稼働率×他の率） |
| **空転・チョコ停ロス** | 設備が動くが非効率 | 性能稼働率 | 供給遅れ、部分停止 | **中**（他の率に乗算） |
| **速度低下ロス** | 設備が動くが遅い | 性能稼働率 | 設計速度を下回る | **中**（性能稼働率に吸収） |
| **不良・手直しロス** | 製品が出ても使えない | 良品率 | 工程内不良、再加工 | **大**（1から直接減算） |
| **立上りロス** | 操業開始の不安定 | 良品率 | 初期段階の不良品 | **中**（良品率に吸収） |

#### ロスの連鎖と改善優先度

```
故障が多い
  ↓
→ 故障ロス：時間稼働率を低下
→ 段取り増加：停止時間がさらに増加
  ↓
→ 急いで製造：速度低下ロス・不良増加
  ↓
→ OEE の全 3 つの率が低下

【改善の効果】
1. 故障削減（予防保全強化）→ 時間稼働率 ↑ → OEE ↑↑↑
2. 段取り削減（段取り替え改善）→ 時間稼働率 ↑ → OEE ↑↑
3. 速度安定化（保全機能向上）→ 性能稼働率 ↑ → OEE ↑
4. 不良削減（品質保全）→ 良品率 ↑ → OEE ↑
```

### 計算例：順算（与えられた各率から OEE を求める）

```
【データ】
  負荷時間 480分、停止時間 60分（故障30分+段取り30分）
  基準サイクルタイム 0.5分/個、加工数量 760個
  不良数量 20個

手順1: 稼働時間
  = 480 − 60 = 420分

手順2: 時間稼働率
  = (480−60) ÷ 480 = 420 ÷ 480 = 0.875（87.5%）

手順3: 性能稼働率
  = (0.5 × 760) ÷ 420 = 380 ÷ 420 ≒ 0.905（90.5%）

手順4: 良品率
  = (760−20) ÷ 760 = 740 ÷ 760 ≒ 0.974（97.4%）

手順5: OEE
  = 0.875 × 0.905 × 0.974
  ≒ 0.771（77.1%）

→ 世界クラスの目標は OEE 85%以上
→ この例では時間稼働率の改善（故障・段取り削減）が最も効果的
```

### 計算例：逆算（OEE 目標から必要な各率を計算）

```
【設定】
  OEE の目標値: 75%以上
  既知の性能稼働率: 98%（ほぼ設計速度で動作）
  既知の良品率: 85%（後工程での再加工を含む）
  求めるもの: 最低限必要な時間稼働率は？

【逆算の手順】
  OEE = 時間稼働率 × 性能稼働率 × 良品率
  0.75 = 時間稼働率 × 0.98 × 0.85

  時間稼働率 = 0.75 ÷ (0.98 × 0.85)
            = 0.75 ÷ 0.833
            ≒ 0.901（約90.1%以上必要）

【解釈】
  → 故障・段取り・調整による停止時間を最大10%程度に抑える必要がある
  → 現在の時間稼働率が90%を下回っていれば、改善の優先順位は「故障削減」
  → 性能稼働率や良品率の余地が小さいため、停止時間削減が最重要
```

## MTBF・MTTR・可用率

### 公式

$$
MTBF = \frac{\text{総稼働時間}}{\text{故障回数}}
$$

$$
MTTR = \frac{\text{総修復時間}}{\text{故障回数}}
$$

$$
\text{可用率（アベイラビリティ）} = \frac{MTBF}{MTBF + MTTR}
$$

| 指標 | 意味 | 大きいほど |
| --- | --- | --- |
| MTBF | 平均故障間隔（壊れにくさ） | 信頼性が高い |
| MTTR | 平均修復時間（直しやすさ） | 保全性が **低い** |
| 可用率 | 使える時間の割合 | 稼働率が高い |

### 計算例

```
【データ】
  総運用時間 1,000時間、故障 4回、総修復時間 40時間

手順1: 総稼働時間
  = 1,000 − 40 = 960時間

手順2: MTBF
  = 960 ÷ 4 = 240時間

手順3: MTTR
  = 40 ÷ 4 = 10時間

手順4: 可用率
  = 240 ÷ (240+10) = 240 ÷ 250 = 0.96（96%）
```

## システム信頼度 ── 直列と並列

### 直列システム

全ての要素が動いて初めてシステムが動く場合。

$$
R_{\text{直列}} = R_1 \times R_2 \times \cdots \times R_n
$$

### 並列システム

1つでも動いていればシステムが動く場合。

$$
R_{\text{並列}} = 1 - (1-R_1)(1-R_2) \cdots (1-R_n)
$$

### 計算例

```
【データ】
  装置A: 信頼度 0.9、装置B: 信頼度 0.8

【直列の場合（AとBの両方が必要）】
  R = 0.9 × 0.8 = 0.72

【並列の場合（AかBのどちらかが動けばOK）】
  R = 1 − (1−0.9)(1−0.8) = 1 − 0.1×0.2 = 1 − 0.02 = 0.98

【直列+並列の複合例】
  AとBが並列 → その結果とCが直列（C: 信頼度 0.95）
  並列部分: 0.98
  全体: 0.98 × 0.95 = 0.931
```

### k-out-of-n 方式（部分冗長性）

3 台以上の設備がある場合、「全部が必要」ではなく「n 台中 k 台以上動けば OK」という方式があります。

**考え方：** n 個の要素があり、そのうち少なくとも k 個が動いていれば成功。

**例：サーバー 3 台、うち 2 台以上稼働で OK**
- 各サーバーの信頼度: R = 0.9
- 故障確率: 1 − R = 0.1
- 失敗する場合: 3 台すべて故障、または 2 台以上故障
- 成功確率 = 1 台以上が故障（P1 台故障 + P2 台故障 + P3 台故障）

```
【計算方法】
  成功 = （ちょうど0台故障）+ （ちょうど1台故障）+ ...
       = C(3,0)×0.9^3×0.1^0 + C(3,1)×0.9^2×0.1^1
       = 1×0.729×1 + 3×0.81×0.1
       = 0.729 + 0.243
       = 0.972

→ 3台全部が0.9の信頼度でも、「2台以上動く」ことで 97.2% まで向上
```

## 設備投資の経済性計算

新しい設備を導入すべきか、いつ更新すべきかを判断するため、**旧設備のランニングコスト** と **新設備の投資額 + ランニングコスト** を比較します。

### 投資回収期間（Payback Period）

```
【設定】
  旧設備（現在稼働中）
    - 年間ランニングコスト: 500万円（修理、部品交換、燃料など）
    - あと 3 年は使える

  新設備（購入検討中）
    - 初期投資: 2,000万円
    - 年間ランニングコスト: 200万円
    - 耐用年数: 10年

【比較】
  新設備の削減効果 = 500 − 200 = 300万円／年

  投資回収期間 = 2,000万円 ÷ 300万円 = 6.67年 ≒ 7年

【判定】
  新設備の耐用年数が10年で、回収期間が7年なら
  → 回収後 3 年間は純利益（導入推奨）
  ただし旧設備はあと3年なので
  → 最速で導入した場合、年間 300万円 × 3年 = 900万円の純利益
  → 7年の全て の利益を享受するには、今すぐ導入すべき
```

### 設備更新の意思決定フレーム

```
判定項目                        判定基準
──────────────────────────────────────────
1. 投資回収期間                 耐用年数の50%以下 → 導入推奨
2. 故障頻度・コスト             月1回以上の修理 → 更新検討
3. 環境・安全規制               新基準に非適合 → 即更新
4. 競争力・品質                 製品品質が劣後 → 即更新
5. 生産能力                     需要増加に対応不可 → 投資
6. 製品ライフサイクル           製品衰退期 → 投資見送り
```

## 典型的なつまずき

- MTBF と MTTR の意味を逆に覚える → MTBF は **故障間隔**（大きい=壊れにくい）、MTTR は **修復時間**（小さい=直しやすい）
- OEE を単なる稼働率と同じだと思ってしまう → OEE は **3つの率の積**。稼働率が高くても速度低下や不良で下がる
- 設備保全と生産性向上を別々の世界だと考えてしまう → 設備のロス削減が **直接** 生産性向上につながる
- 並列信頼度を「足し算」する → `1−(1−R1)(1−R2)` が正しい
- 可用率の分母を間違える → 分母は `MTBF + MTTR`（全時間のうち稼働割合）

## 問題を解くときの観点

- 問われているのは `止まりにくさ`（MTBF）か `直しやすさ`（MTTR）か `総合効率`（OEE）か
- ロスの原因は停止か、速度か、不良か
- システム構成は直列か、並列か、複合か
- 改善の狙いは設備安定か、作業効率か

## 確認問題

<Callout title="問1：OEE の順算と改善優先度">
    負荷時間 500分、停止時間 50分、基準サイクルタイム 0.4分/個、加工数量 1,000個、不良数量 50個。OEE を求めよ。また、現在の3つの率のうち、改善すべき優先順位を述べよ。

    **解答**：時間稼働率 = (500−50)÷500 = 0.90。性能稼働率 = (0.4×1000)÷450 = 400÷450 ≒ 0.889。良品率 = (1000−50)÷1000 = 0.95。OEE = 0.90×0.889×0.95 ≒ 0.760（76.0%）。優先順位：1位 性能稼働率（88.9%、最も低い）→ 速度低下・チョコ停ロス削減。2位 時間稼働率（90%）→ 故障・段取り削減。3位 良品率（95%）→ 既に高い。
</Callout>

<Callout title="問2：OEE の逆算（目標から必要な率を求める）">
    OEE の目標を 82% 以上にしたい。現在の性能稼働率が 92%、良品率が 96% の場合、最低限必要な時間稼働率は何%か。

    **解答**：OEE = 時間稼働率 × 0.92 × 0.96。0.82 = 時間稼働率 × 0.8832。時間稼働率 = 0.82 ÷ 0.8832 ≒ 0.928（92.8% 以上必要）。つまり停止時間をさらに削減し、負荷時間に対する稼働率を 92.8% まで上げる必要がある。
</Callout>

<Callout title="問3：可用率の計算と MTBF・MTTR の関係">
    MTBF = 200時間、MTTR = 8時間。可用率を求めよ。また、MTTR を 4時間に短縮した場合、可用率の改善は何ポイントか。

    **解答**：可用率 = 200÷(200+8) = 200÷208 ≒ 0.962（96.2%）。MTTR短縮後：204÷204 ≒ 0.980（98.0%）。改善：98.0% − 96.2% = 1.8ポイント。MTBF を大幅に伸ばすより、MTTR（直しやすさ）を小さく改善する方が、時間・コストで効率的な場合も多い。
</Callout>

<Callout title="問4：複合システムの信頼度（直列+並列）">
    装置A（信頼度0.95）と装置B（信頼度0.90）が並列に接続され、その結果と装置C（信頼度0.85）が直列に接続されている。システム全体の信頼度を求めよ。

    **解答**：並列部分 = 1−(1−0.95)(1−0.90) = 1−0.05×0.10 = 1−0.005 = 0.995。全体 = 0.995×0.85 ≒ 0.846。C が全体を左右する。
</Callout>

<Callout title="問5：k-out-of-n 方式（3 台中 2 台以上稼働）">
    ポンプ 3 台を並列配置し、うち 2 台以上が稼働すれば良い。各ポンプの信頼度が 0.9 の場合、システム全体の信頼度を求めよ。

    **解答**：全て故障しない OR 1台故障する確率。全て正常：0.9^3 = 0.729。ちょうど1台故障：C(3,1)×0.9^2×0.1 = 3×0.81×0.1 = 0.243。合計 = 0.729 + 0.243 = 0.972（97.2%）。冗長性により、個別信頼度 90% でも、システム信頼度 97% を達成できる。
</Callout>

<Callout title="問6：設備投資の回収期間">
    旧設備のランニングコストは年 600万円。新設備の初期投資 3,000万円、年間ランニングコスト 150万円の場合、投資回収期間は何年か。また、新設備の耐用年数が 12 年なら、導入は経済的か。

    **解答**：年間削減額 = 600 − 150 = 450万円。回収期間 = 3,000 ÷ 450 = 6.67年 ≒ 6.7年。耐用年数 12 年 > 回収期間 6.7 年 なので、回収後 5.3 年間は純利益。導入は経済的に推奨される。ただし故障頻度や環境規制なども合わせて検討すること。
</Callout>
