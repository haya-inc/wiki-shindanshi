## このページの役割

<Callout title="このページの役割">
    このページは、`必要なものを必要なときに必要なだけ流す` という JIT の考え方を整理する解説ページです。
    引っ張る管理の代表として、かんばん方式とセットで押さえます。
</Callout>

<Callout title="このページの読み方">
    MRP が「計画を立てて前から押し出す」方式なら、JIT は「後工程が必要な分だけ引き取る」方式です。どちらが正しいかではなく、前提条件（需要の安定度、品種数、段取り時間）に応じてどちらが合うかを判断できるようにしましょう。
</Callout>

- `プル型` と `プッシュ型` の違いが最重要

- JIT の考え方を説明できるか

- 7つのムダを列挙できるか

## JIT（Just In Time）

JIT は、必要なものを必要なときに必要なだけ生産・調達する考え方です。トヨタ生産方式（TPS）の 2 本柱の 1 つ（もう 1 つは自働化）。

### JIT の前提条件

| 前提 | 内容 |
| --- | --- | --- |
| 平準化生産 | 品種と量を均等に配分（急な偏りをなくす） |
| 段取り替え時間の短縮 | 小ロット化のために段取り時間を極限まで減らす |
| 多能工化 | 1人が複数工程を担当できる体制 |
| 工程の流れ化 | 工程間の在庫を最小化する配置 |

#### 段取り替え時間の短縮（SMED）

段取り替え時間が長いと、小ロット化ができず、在庫が増えてしまいます。トヨタが開発した SMED（Single Minute Exchange of Die）は、段取り時間を1桁分（数分以内）に短縮する手法です。

**SMED の4ステップ**

1. **内段取りと外段取りを区別する**
   - 外段取り：機械を停止せずにできる準備（金型の準備、部品の搬出など）
   - 内段取り：機械を停止して行う作業（金型の交換、調整など）

2. **外段取り化**
   - 内段取りのうち、実は外でできる作業を洗い出す
   - 例：金型をあらかじめ準備、工具をそばに置く

3. **内段取りの短縮**
   - 内段取りだけになった作業をさらに削減
   - 例：調整の簡略化、ボルトを減らす

4. **全体の最適化**
   - 人員配置を工夫して、外段取り中に他の作業を進める

##### 段取り短縮の効果例

とある板金工場の事例：

| 段階 | 段取り時間 | 効果 |
| --- | --- | --- |
| 改善前 | 60分 | 1日3ロット |
| 外段取り化 | 30分 | 1日6ロット |
| 内段取り短縮 | 15分 | 1日12ロット（日当たり在庫圧縮） |
| さらに改善 | 5分 | ほぼシングル品目運用可能 |

この短縮により、品種切替が容易になり、小ロット・多品種生産が実現します。

#### 平準化生産（レベリング）

需要が不安定だと、かんばん方式が成立しません。そこで、月間需要を日単位に「平準化」して、毎日同じ量・品種の生産を目指します。

##### 平準化の計算例

あるメーカーの月間需要：

| 製品 | 月間需要量 | 日当たり需要（20日稼働） |
| --- | --- | --- |
| 製品A | 600個 | 30個/日 |
| 製品B | 300個 | 15個/日 |
| 製品C | 100個 | 5個/日 |
| **合計** | **1,000個** | **50個/日** |

**平準化した生産順序の例**

毎日、以下のパターンで生産を循環させます：

```
A → A → B → A → A → B → A → C → (繰り返し)
```

この順序なら：
- 朝の1時間で A-A-B を完成させ、かんばんが戻ってくる
- B の在庫不足、C の過剰を避ける
- 需要変動に素早く対応できる

**平準化の効果**

- 在庫が安定 → かんばん枚数が最小化
- 作業者の手待ちが減る
- 不良品の原因となるリズムの乱れが減る

### 自働化（にんべんのある自動化）

異常が発生したら機械が自動で停止し、人に知らせる仕組みです。不良品が次工程に流れることを防ぎ、問題を即座に顕在化させます。

## かんばん方式

かんばん方式は、後工程が前工程から必要な量だけ引き取る `プル型` の生産管理です。かんばん（カード）が情報伝達の媒体となります。

### かんばんの種類

| 種類 | 役割 | 流れ |
| --- | --- | --- |
| 引取りかんばん | 後工程→前工程への引取り指示 | 後工程が前工程のストアから部品を引き取る |
| 仕掛けかんばん（生産指示かんばん） | 前工程への生産指示 | 引き取られた分だけ前工程が生産する |

### かんばんの運用ルール

```
① 後工程引取り: 後工程は必要なものを必要な量だけ前工程から引き取る
② 前工程は引き取られた分だけ生産する（作りすぎない）
③ 不良品を次工程に送らない
④ かんばんの枚数を最小限にする（在庫圧縮の手段）
⑤ かんばんは微調整に使う（大きな変動には対応できない）
```

### かんばん枚数の計算

かんばんの最適枚数は、安全在庫を確保しながら、できるだけ少なく設定します。

$$N = \frac{D \times L \times (1 + \alpha)}{a}$$

ここで：
- **N**: かんばん枚数
- **D**: 日当たり需要量（個/日）
- **L**: リードタイム（日）
- **α**: 安全係数（0.1～0.2 程度）
- **a**: 1枚あたりの収容数（個）

#### 計算例

自動車部品Aの場合：
- 日当たり需要量 D = 100個/日
- リードタイム L = 0.5日（12時間）
- 安全係数 α = 0.1（10%）
- 1枚あたり収容数 a = 20個

$$N = \frac{100 \times 0.5 \times 1.1}{20} = \frac{55}{20} = 2.75$$

→ **3枚** に設定。これで最大在庫は 3 × 20 = 60個になります。

#### 運用のコツ

- 枚数が多い → リードタイムが長い、または需要変動が大きい可能性
- 枚数を減らす → 段取り短縮でリードタイムを短縮、平準化で需要を安定化
- 季節変動時は α（安全係数）を一時的に上げて対応

## プル型 vs プッシュ型

| 比較軸 | プル型（JIT/かんばん） | プッシュ型（MRP） |
| --- | --- | --- |
| 起点 | 後工程の消費（実需） | 生産計画（予測需要） |
| 情報の流れ | 後→前（引き取り） | 前→後（計画展開） |
| 在庫の考え方 | 最小化（ムダ） | 必要量を計算して保持 |
| 変動への対応 | 小さな変動に強い | 大きな変動に対応可能 |
| 前提 | 平準化生産、段取り短縮 | BOM、リードタイム、在庫データ |
| 代表的な業種 | 自動車（トヨタ） | 組立型製造業全般 |

<Callout title="プルとプッシュの使い分け">
    JIT とMRP は対立概念ではなく、現場の特性に応じて使い分けます。需要が安定し平準化できる環境ではプル型、品種や需要変動が大きい環境ではプッシュ型が向きます。実際には両者を組み合わせるハイブリッドも多いです。
</Callout>

### プル型とプッシュ型の判断基準

どちらを採用すべきかは、以下の4つの条件で判断します。

| 判断基準 | プル型（JIT）が向く | プッシュ型（MRP）が向く |
| --- | --- | --- |
| 需要の安定度 | 月ごと、日ごとの変動が小さい | 季節変動、不規則変動が大きい |
| リードタイム | 短い（数日～1日以内） | 長い（数週間以上） |
| 品種数 | 少ない～中程度（10～50品種） | 多い（100品種以上） |
| 段取り時間 | 短い（1時間以内）| 長い（数時間以上） |

#### 判断フローチャート

```
Q1: 需要は月ごと・日ごとで安定しているか？
  → NO → プッシュ型（MRP）を検討
  → YES → Q2へ

Q2: リードタイムは 1日以内に短縮できるか？
  → NO → プッシュ型（MRP）を検討
  → YES → Q3へ

Q3: 段取り替え時間は 1時間以内か？
  → NO → 段取り短縮（SMED）を先に実施
  → YES → Q4へ

Q4: 品種数は 50品種以下か？
  → NO → ハイブリッド型（主流品はプル、少量品はプッシュ）
  → YES → プル型（JIT + かんばん）で運用可能
```

### ハイブリッド型の実例

実際の自動車工場では以下のような使い分けをしています：

- **主流品種**（生産量80%）→ プル型で日単位で引き取り
- **少量品種**（生産量20%）→ プッシュ型で月単位で計画生産
- **部品レベル** → 標準部品はプル、特注部品はプッシュ

## 7つのムダ

トヨタ生産方式では、以下の 7 つを排除すべきムダとして定義しています。

| ムダ | 内容 | 例 |
| --- | --- | --- |
| 作りすぎのムダ | 必要以上に生産する | 余剰在庫の発生 |
| 手待ちのムダ | 作業者が何もできない時間 | 前工程待ち |
| 運搬のムダ | 不必要な移動 | レイアウト不良による長距離搬送 |
| 加工そのもののムダ | 不要な加工や過剰品質 | 顧客要求を超える仕上げ |
| 在庫のムダ | 必要以上の在庫保持 | 倉庫の圧迫、資金の固定化 |
| 動作のムダ | 不必要な人の動き | 工具を取りに行く |
| 不良をつくるムダ | 不良品の発生 | 手直し、廃棄 |

最も重いムダは `作りすぎのムダ` です。これが他の6つのムダ（在庫、運搬、手待ちなど）を生み出す根本原因となります。

### 「作りすぎのムダ」が他のムダを生み出すメカニズム

あるメーカーが月間1000個の需要なのに、毎月1500個を生産している場合を考えましょう。

```
作りすぎ（500個/月）
  ↓
在庫のムダ → 倉庫費用、資金固定化、陳腐化リスク
  ↓
運搬のムダ → 倉庫への運搬、回転運搬、廃棄運搬
  ↓
手待ちのムダ → 在庫が多いと後工程の作業者は「次の部品を待つ」のではなく「不要な部品で詰まる」
  ↓
動作のムダ → 在庫を整理する、古い部品を探す
  ↓
不良をつくるムダ → 古い在庫から不良品が見つかる、陳腐化品の手直し
```

**つまり、作りすぎを防ぐことが、他の6つのムダを連鎖的に防ぐ最善の方法です。** これが JIT の哲学です。

## 標準作業

標準作業は、品質・効率・安全を保つために定めた最善の作業方法です。JIT の安定運用には標準化が不可欠です。

### 標準作業の3要素

| 要素 | 内容 |
| --- | --- |
| タクトタイム | 1個を何秒（何分）で作るべきか |
| 作業順序 | どの順で作業するか |
| 標準手持ち | 工程間に最低限必要な仕掛品の量 |

#### タクトタイムの計算

タクトタイムは、需要に合わせて生産ペースを決める基準です。

$$\text{タクトタイム} = \frac{\text{稼働時間（分）}}{\text{必要生産数（個）}}$$

##### 計算例

製造工場の1シフト（8時間）で480個の製品を作る場合：

- 稼働時間：8時間 = 480分
- 必要生産数：480個

$$\text{タクトタイム} = \frac{480}{480} = 1 \text{分/個}$$

つまり、**1個を1分で作らなければならない** ということです。この場合：
- 1時間（60分）で60個を生産できる
- 遅くなると全体の遅延につながる
- 作業者は60秒以内に全工程を完了させる必要がある

#### タクトタイムと段取り替え時間の関係

- タクトタイムが短い（1分/個）→ 段取り替え時間は数分以内に（そうしないと引き取りに応じられない）
- タクトタイムが長い（30分/個）→ 段取り替え時間に余裕がある

JIT では、タクトタイムを短くする（=高速化）のではなく、**需要に合わせて段取りと手待ちを最小化** することを重視します。

#### 標準手持ち（在庫）の考え方

- 各工程間に「最低限」の仕掛品だけを保つ
- 工程A → 待ち → 工程B という流れで、待ち時間のムダを減らす
- 標準手持ちは、リードタイムと需要変動から逆算して決定

## 典型的なつまずき

- JIT を単なる在庫削減手法だと思ってしまう → JIT は **流れの最適化**。在庫削減は結果
- かんばん方式を伝票処理の仕組みだと誤解する → かんばんは **生産と引取りの指示** を出す情報伝達ツール
- MRP と JIT の違いを「古い/新しい」で覚える → **プッシュ/プル** の違い。どちらが優れているかではなく前提条件が異なる
- 7つのムダを一覧暗記で終える → `作りすぎのムダ` が **根本原因** であることを理解する
- 平準化をJITのオプションだと思う → 平準化は JIT の **必須前提**

## 問題を解くときの観点

- 問われているのは `流れ` か `計画展開` か
- 引っ張る管理（プル）か、押し出す管理（プッシュ）か
- 在庫を減らしたいのか、流れを安定させたいのか
- 前提条件（平準化、段取り短縮）が整っているか

## 確認問題

<Callout title="問1：プル型とプッシュ型">
    「後工程が使った分だけ前工程に生産を指示する」方式は、プル型・プッシュ型のどちらか。また、この方式を実現する代表的な仕組みは何か。

    **解答**：プル型。代表的な仕組みは「かんばん方式」。後工程の消費を起点に前工程が生産するため、作りすぎのムダを防止できる。
</Callout>

<Callout title="問2：7つのムダ">
    トヨタ生産方式で「最も重いムダ」とされるのはどれか。また、その理由を説明せよ。

    **解答**：作りすぎのムダ。必要以上に生産すると、在庫のムダ（保管コスト）、運搬のムダ（移動）、手待ちのムダ（後工程が不要な部品で詰まる）など、他の6つのムダを連鎖的に生み出すため。
</Callout>

<Callout title="問3：JIT の前提条件">
    JIT を導入するために整えるべき前提条件を 3 つ挙げよ。

    **解答**：①平準化生産（品種と量の均等配分）、②段取り替え時間の短縮（小ロット化の実現）、③多能工化（1人が複数工程を担当）。これらが整わないと在庫を持たない運用は破綻する。
</Callout>
