## このページの役割

<Callout title="このページの役割">
    このページは、`組織の形をどう切るか` を整理する解説ページです。`機能別組織`、`事業部制組織`、
    `マトリクス組織` を中心に、`誰が意思決定するか`、`どこで調整が詰まるか`、`どんな環境に合うか`
    を比較できるようにします。
</Callout>

<Callout title="このページの読み方">
  「なぜ事業部制に変えたのか？」──事例Ⅰで頻出のテーマです。組織形態の選択は環境・戦略・規模で決まります。各形態のメリット/デメリットだけでなく、「どの条件でどの形態が合うか」を判断できるようにしましょう。
</Callout>

## 学習のポイント

- まず `分業の基準` を見る（職能か事業か）
- 次に `権限の置き方` を見る（集権か分権か）
- 最後に `環境への適合` を見る（機械的か有機的か）

## 試験で何が問われるか

- 機能別組織と事業部制組織の違いを比較表で説明できるか
- マトリクス組織の「命令の二元性」を見抜けるか
- コンティンジェンシー理論の各研究者を区別できるか
- ミンツバーグの組織コンフィギュレーションを説明できるか

## 組織形態の比較（試験最頻出）

| 項目 | 機能別組織 | 事業部制組織 |
| --- | --- | --- |
| 分業の基準 | 職能（営業・製造等） | 製品・地域・顧客 |
| 意思決定 | 集権的（トップが判断） | 分権的（事業部長が判断） |
| 利益責任 | トップが一括管理 | 各事業部が利益責任を持つ |
| 適する環境 | 安定的・単一事業 | 不確実・多角化企業 |
| 規模の経済 | 発揮しやすい | 発揮しにくい（重複） |
| 経営者育成 | 困難（専門家育成） | 容易（事業部長がミニ経営者） |
| 部門間の調整 | 必要性高い（トップが調整） | 事業部内で完結しやすい |

### その他の組織形態

| 形態 | 特徴 | メリット | デメリット |
| --- | --- | --- | --- |
| マトリクス組織 | 機能軸と事業軸の二重構造 | 横断連携、柔軟性 | **命令の二元性**、コンフリクト |
| カンパニー制 | 事業部の独立性をさらに高めた擬似分社化 | 高い自律性 | 全社最適困難 |
| プロジェクト組織 | 特定目的の一時的組織 | 部門横断の連携 | 終了後の再配置問題 |
| ネットワーク組織 | 企業間の緩やかな連携 | 柔軟性、低コスト | 統制困難 |
| 持株会社 | 傘下企業の株式を保有して統括 | グループ戦略に集中 | 遠心力による統制低下 |

## 組織設計の原則

| 原則 | 内容 |
| --- | --- |
| 専門化の原則 | 業務を分業して専門性を高める |
| 権限・責任一致の原則 | 権限と責任は同等であるべき |
| 統制範囲の原則 | 1人が管理できる部下の数には限界がある |
| 命令統一性の原則 | 1人の部下は1人の上司からのみ命令を受ける |
| 例外の原則 | 定型業務は下位に委譲し、上位は例外事項のみ |

---

## 集権と分権

| 項目 | 集権的組織 | 分権的組織 |
| --- | --- | --- |
| 意思決定権限 | 上位に集中 | 下位に委譲 |
| メリット | 統一的方針、統制容易 | 迅速な意思決定、現場適応 |
| デメリット | トップの過負荷、現場対応の遅れ | 全体最適困難、方針のブレ |
| 適する状況 | 安定環境、規模が小さい | 不確実環境、多角化、大規模 |

---

## 機械的組織と有機的組織（バーンズ＆ストーカー）

| 項目 | 機械的組織 | 有機的組織 |
| --- | --- | --- |
| 環境 | 安定 | 不確実 |
| 構造 | 公式的・階層的 | 柔軟・フラット |
| コミュニケーション | 垂直的 | 水平的 |
| ルール | 厳格 | 緩やか |
| 意思決定 | トップダウン | 現場分散 |

---

## コンティンジェンシー理論（条件適合理論）

唯一最善の組織はなく、環境条件に適合した組織が最も有効です。

| 研究者 | 条件変数 | 内容 |
| --- | --- | --- |
| バーンズ＆ストーカー | 環境の安定性 | 安定→機械的、不安定→有機的 |
| ウッドワード | 技術の複雑性 | 単品→有機的、大量→機械的、装置→有機的 |
| ローレンス＆ローシュ | 環境の不確実性 | 不確実性高→分化と統合の必要性 |
| ガルブレイス | 情報処理 | 不確実性高→情報処理量増大→組織設計で対処 |

## ミンツバーグの組織構造論

### 組織の5つの基本部分

| 基本部分 | 役割 |
| --- | --- |
| 戦略的頂点 | トップマネジメント |
| ミドルライン | 中間管理職 |
| オペレーティング・コア | 現場の作業者 |
| テクノストラクチャー | 分析スタッフ（業務の標準化） |
| サポートスタッフ | 間接的な支援（法務・広報等） |

### 5つの組織コンフィギュレーション

| 類型 | 主要な調整メカニズム | 中心部分 |
| --- | --- | --- |
| 単純構造 | 直接の監督 | 戦略的頂点 |
| 機械的官僚制 | 作業プロセスの標準化 | テクノストラクチャー |
| 専門職官僚制 | 技能の標準化 | オペレーティング・コア |
| 事業部制組織 | アウトプットの標準化 | ミドルライン |
| アドホクラシー | 相互調整 | サポートスタッフ |

## 組織のライフサイクル（グレイナー）

| 段階 | 成長の原動力 | 直面する危機 |
| --- | --- | --- |
| 第1段階 | 創造性 | リーダーシップの危機 |
| 第2段階 | 指揮 | 自主性の危機 |
| 第3段階 | 委譲 | 統制の危機 |
| 第4段階 | 調整 | 形式主義の危機 |
| 第5段階 | 協働 | （次の危機へ） |

## 典型的なつまずき

- 機能別と事業部制を組織図の見た目だけで覚える → **意思決定の集権/分権** と **利益責任の所在** で区別
- マトリクス組織を万能だと思う → **命令の二元性** が最大のデメリット
- 集権を悪、分権を善と単純化する → 環境と戦略に応じた **使い分け**
- コンティンジェンシー理論の研究者を混同する → 条件変数（環境/技術/情報処理）で区別
- グレイナーの「危機」を失敗と捉える → 成長に伴う **自然な課題** であり、克服して次の段階へ進む

## 問題を解くときの観点

- 分業の基準は `職能` か `事業` か
- 利益責任をどこに置いているか
- 調整負荷は `トップ` `事業部内` `横断チーム` のどこに出るか
- 環境の不確実性に対して、統制と柔軟性のどちらを優先しているか

## 確認問題

<Callout title="問1：組織形態の選択">
    「単一事業で安定した環境のもと、営業・製造・経理の各部門が専門性を発揮している」組織はどの類型か。

    **解答**：機能別組織。単一事業・安定環境で、職能別に分業して専門性と規模の経済を追求する形態。
</Callout>

<Callout title="問2：命令の二元性">
    「ある技術者が、機能部門の上司と製品プロジェクトのリーダーの双方から異なる指示を受けている」。これはどの組織形態で起きやすい問題か。

    **解答**：マトリクス組織。機能軸と事業軸の二重構造により、命令の二元性が生じる。命令統一性の原則に反するため、コンフリクトが発生しやすい。
</Callout>

<Callout title="問3：グレイナーモデル">
    「急成長した企業で、創業者の個人的判断だけでは組織を統率できなくなった」。グレイナーモデルのどの危機にあたるか。

    **解答**：第1段階のリーダーシップの危機。創造性による成長の後、組織が大きくなり創業者の限界が顕在化する。次の段階では「指揮」による成長（管理体制の構築）が必要。
</Callout>
