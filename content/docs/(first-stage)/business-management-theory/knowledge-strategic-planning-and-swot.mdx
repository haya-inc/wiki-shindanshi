## このページの役割

<Callout title="このページの役割">
    このページは、`経営理念`、`ビジョン`、`ドメイン`、`SWOT`、`計画策定` の流れを最初に固めるための解説ページです。
    `何を目指すか` と `どこで戦うか` を切り分けられる状態を目指します。
</Callout>

## 学習のポイント

- `理念` は存在意義、`ビジョン` は将来像、`ドメイン` は事業領域として分ける
- `外部環境` と `内部資源` を混ぜずに SWOT へ置く
- 分析した結果を `目標` と `施策` に落とす流れまでつなげる

## 試験で何が問われるか

- 経営理念、ビジョン、ドメインの違いを説明できるか
- SWOT の `機会 / 脅威` と `強み / 弱み` を正しく置けるか
- 分析結果を受けて、どんな戦略課題や施策が自然かを判断できるか

## 定義と仕組み

### 経営理念、ビジョン、ドメイン

`経営理念` は企業の存在意義です。`ビジョン` は将来どうありたいかという姿で、`ドメイン` は誰に何を提供するかという事業領域です。

この 3 つは似ていますが、役割が違います。理念が `なぜ存在するか`、ビジョンが `どこへ向かうか`、ドメインが `どこで戦うか` です。

### SWOT 分析

`SWOT` は、`機会` と `脅威` を外部環境、`強み` と `弱み` を内部資源として整理する枠組みです。大事なのは、`社内の話` と `市場や競争の話` を混ぜないことです。

### 計画策定の流れ

戦略論の入口では、`理念 / ビジョン` を確認し、`環境分析` を行い、`課題` を定め、`目標` と `施策` に落とします。分析だけで終わらず、どの打ち手へつながるかまで読むことが重要です。

## 典型例

| 論点 | 見ること | 典型的な使い方 |
| --- | --- | --- |
| 経営理念 | 企業の存在意義 | 方向性の一貫性を確認する |
| ドメイン | 顧客、機能、技術 | どの市場で戦うかを定める |
| SWOT | 外部環境と内部資源 | 戦略課題を整理する |
| 計画策定 | 目標、施策、資源配分 | 分析結果を行動へ落とす |

## 典型的なつまずき

- `市場成長率` を `強み` に入れてしまう
- `技術力` を `機会` のように扱ってしまう
- 理念、ビジョン、ドメインを全部 `経営方針` として一括で覚えてしまう
- SWOT を書いて終わり、どんな施策につながるかを考えない

## 問題を解くときの観点

- いま見ているのは `社外` か `社内` か
- 問われているのは `存在意義`、`将来像`、`事業領域` のどれか
- SWOT の結果から `何を伸ばし、何を補うか` が言えるか
