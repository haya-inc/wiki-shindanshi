## このページの役割

このページは、`Webテクノロジーとクラウド` を整理する知識ノードです。`ネットワークの上で何を使うか` と `どこまでをサービスとして借りるか` を切り分けます。

## 学習のポイント

- まず `Web の基本構造` を押さえる
- 次に `HTTP / HTTPS` と `API` の役割を切る
- そのうえで `SaaS / PaaS / IaaS` を利用範囲で比較する

## 試験で何が問われるか

- `Web サービス` と `API` の違いを言えるか
- `HTTP` と `HTTPS` の違いを説明できるか
- `SaaS`、`PaaS`、`IaaS` を利用者が何を持つかで比較できるか
- `オンプレミス` と `クラウド` の違いを言えるか

## 定義と仕組み

### Web の基本構造

Web は、ブラウザなどの利用者側と、サーバ側がやり取りして情報を表示する仕組みです。`クライアント` と `サーバ` の役割を分けて押さえると、後の API やクラウドも読みやすくなります。

### HTTP / HTTPS

`HTTP` は Web 上で情報をやり取りするための基本的な約束です。`HTTPS` は、その通信を暗号化して安全性を高めたものです。したがって、`Web で使う約束` と `その安全化` を分けます。

### API

`API` は、別のシステムやサービスと機能やデータをやり取りするための接点です。画面を人が見るための Web ページと、機械どうしが使う API は役割が違います。

### クラウドのサービスモデル

`SaaS` は完成したソフトウェアを使う形、`PaaS` はアプリを動かす土台を借りる形、`IaaS` は仮想サーバやストレージのような資源を借りる形です。利用者がどこまで管理するかで切り分けます。

## 典型例

- `Web会議サービス` や `グループウェア` は SaaS
- `アプリ実行基盤` を借りるのは PaaS
- `仮想サーバ` や `ストレージ` を借りるのは IaaS
- EC サイトが外部決済とつながるのは API 連携

## 典型的なつまずき

- `HTTP` と `HTML` を同じ層の話として混ぜる
- `Webページ` と `API` の違いを曖昧にする
- `SaaS / PaaS / IaaS` を語感だけで暗記する
- クラウドとアウトソーシングの違いを整理しない

## 問題を解くときの観点

- 問われているのは `見る仕組み` か `つなぐ仕組み` か
- そのサービスは `完成品` を使うのか `土台` を借りるのか `資源` を借りるのか
- 利用者側に何が残るか
- セキュリティの話なら `HTTPS` や認証とどうつながるか
