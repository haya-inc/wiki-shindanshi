## このページの役割

このページは、`データベースとSQL` を整理する知識ノードです。`表の関係`、`整えるルール`、`取り出す命令` を 1 本の流れで押さえます。

## 学習のポイント

- まず `表` と `キー` の関係を理解する
- 次に `正規化` で重複と更新異常を防ぐ考え方を押さえる
- そのうえで `SELECT`、`JOIN`、`GROUP BY`、`トランザクション` を読む

## 試験で何が問われるか

- `主キー` と `外部キー` の違いを言えるか
- `正規化` の目的を説明できるか
- 基本的な `SQL` を表の関係と一緒に読めるか
- `トランザクション` と `排他制御` の役割を言えるか

## 定義と仕組み

### リレーショナルデータベース

`RDB` は、行と列で表した表を関係付けて管理する仕組みです。1つの大きな表へ全部詰め込むより、意味のまとまりごとに表を分け、キーでつなぐ方が扱いやすくなります。

### キー

`主キー` は行を一意に識別する項目です。`外部キー` は別の表の主キーを参照し、表どうしを結び付けます。したがって、`誰が誰とつながるか` を追うのが SQL 問題の入口です。

### 正規化

正規化は、重複を減らし、更新異常や削除異常を防ぐ考え方です。試験では細かな定義よりも、`同じ情報を何度も持たないように分ける` という目的を押さえる方が重要です。

### SQL

`SELECT` は取り出す、`WHERE` は条件を絞る、`JOIN` は表をつなぐ、`GROUP BY` は集計する、という役割で読みます。文法暗記だけでなく、`どの表から何を引くか` を先に考えます。

### トランザクション

トランザクションは、一連の処理を `まとめて成功` させるか `まとめて戻す` かを扱う考え方です。整合性を保つために、`コミット` と `ロールバック`、`排他制御` が関わります。

## 典型例

- 顧客表と受注表を `顧客ID` で結ぶ
- `売上が一定以上の顧客` を `WHERE` で抽出する
- `商品別売上高` を `GROUP BY` で集計する
- 在庫更新と売上計上を 1 つのトランザクションとして扱う

## 典型的なつまずき

- `主キー` と `外部キー` の役割が混ざる
- 正規化を `表を増やすこと` だとだけ覚えてしまう
- SQL を日本語にせず、文法だけで読もうとする
- `JOIN` でどの表どうしをつなぐのか見失う

## 問題を解くときの観点

- まず `どの表があるか` を確認する
- 次に `どのキーでつながるか` を見る
- そのあとで `抽出` なのか `集計` なのかを決める
- トランザクションなら `途中で止まると困る処理か` を考える
