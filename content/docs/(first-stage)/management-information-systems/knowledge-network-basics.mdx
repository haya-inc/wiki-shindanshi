## このページの役割

このページは、`通信ネットワーク` を整理する知識ノードです。`どうつなぐか`、`どこへ届けるか`、`名前をどう解決するか` を順に押さえます。

## 学習のポイント

- まず `層` の考え方を押さえる
- 次に `TCP/IP` と `IPアドレス` の役割を切る
- そのうえで `DNS`、`LAN / WAN`、`ルータ` の位置づけを整理する

## 試験で何が問われるか

- `OSI 参照モデル` と `TCP/IP` の大まかな対応を説明できるか
- `IPアドレス`、`サブネット`、`ルータ` の役割を言えるか
- `DNS` が何をしている仕組みか説明できるか
- `LAN`、`WAN`、`無線LAN` の違いを言えるか

## 定義と仕組み

### 層の考え方

通信は、役割ごとに層を分けて考えると整理しやすくなります。試験では厳密な全層暗記より、`どの層で何をしているか` を大まかに言えることが大切です。

### TCP/IP

`IP` は届け先を決める仕組み、`TCP` は順序や欠落を確認しながら確実に届ける仕組みです。したがって、`どこへ送るか` と `どう安定して送るか` を分けて覚えます。

### IPアドレスとサブネット

`IPアドレス` は機器を識別する番号です。`サブネット` はネットワークを分けて管理しやすくする考え方で、同じネットワーク内かどうかを判断するときに使います。

### DNS

`DNS` は、ドメイン名を IP アドレスへ対応付ける仕組みです。利用者は名前でアクセスし、裏側でアドレスへ変換されます。

### LAN / WAN

`LAN` は狭い範囲のネットワーク、`WAN` は広い範囲をまたぐネットワークです。企業では拠点内は LAN、拠点間は WAN という見方で整理すると分かりやすくなります。

## 典型例

- `192.168...` のようなアドレスは機器の識別番号
- `example.com` のような名前は DNS で IP アドレスへ変換する
- 社内ネットワークは LAN、複数拠点接続は WAN
- パケットを別ネットワークへ中継するのがルータ

## 典型的なつまずき

- `IPアドレス` と `ドメイン名` を同じものとして扱う
- `TCP` と `IP` をまとめて 1 つの役割だと覚える
- `LAN` と `WAN` の違いを距離感だけで曖昧に覚える
- 層の話を丸暗記し、実際に何をしているか言えない

## 問題を解くときの観点

- 問われているのは `識別`、`配送`、`名前解決`、`中継` のどれか
- その技術は `同じネットワーク内` の話か `別ネットワークへ渡す` 話か
- 利用者が見ているのは `名前` か `アドレス` か
- `どこで何を担う層か` を日本語で言えるか
