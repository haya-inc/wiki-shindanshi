## このページの役割

このページは、`コンピュータの基礎` を整理する知識ノードです。`どの装置が何をするか`、`どのソフトウェアがどこに位置するか` を切り分けます。

## 学習のポイント

- まず `ハードウェア` と `ソフトウェア` を分ける
- 次に `CPU / メモリ / 補助記憶装置` の役割を切る
- そのうえで `OS / ミドルウェア / アプリケーション` の層を固定する

## 試験で何が問われるか

- `CPU`、`主記憶`、`補助記憶` の役割を説明できるか
- `OS` と `ミドルウェア` の違いを言えるか
- `集中処理`、`分散処理`、`クライアントサーバ`、`仮想化` の位置づけを分けられるか
- `RASIS`、`RAID`、`負荷分散` を、止めない仕組みとして読めるか

## 定義と仕組み

### ハードウェア

`CPU` は命令を実行する中枢です。`主記憶` は実行中データを一時的に置く場所、`補助記憶装置` は長期保存の場所です。したがって、`速いが揮発する` と `遅いが残る` を分けて覚えます。

### ソフトウェア

`OS` は資源管理と入出力制御を担い、`ミドルウェア` は DBMS や Web サーバのように OS とアプリケーションの間を支えます。`アプリケーション` は業務や利用目的に直接使うソフトウェアです。

### 処理形態

`集中処理` は一か所で処理を集約し、`分散処理` は複数の機器へ分けます。`クライアントサーバ` は役割分担の形、`仮想化` は 1 台の資源を論理的に分けて使う考え方です。

### 信頼性と停止対策

`RAID`、`冗長化`、`負荷分散`、`フォールトトレランス` は、故障しても止まりにくくする仕組みです。`RASIS` は、何を良くしたい指標かをまとめて見る枠組みとして使います。

## 典型例

- 処理速度を上げる話では `CPU` や `メモリ`
- データ保存やバックアップの話では `補助記憶装置`
- DB や Web サービスの土台の話では `OS` と `ミドルウェア`
- 障害に強い構成の話では `RAID`、`冗長化`、`負荷分散`

## 典型的なつまずき

- `メモリ` と `補助記憶装置` をどちらも保存装置として雑に覚える
- `OS` と `ミドルウェア` の境界を曖昧にする
- `クライアントサーバ` と `分散処理` を同じ概念として扱う
- `RAID` や `負荷分散` を、何を守る仕組みか分けずに暗記する

## 問題を解くときの観点

- 問われているのは `処理`、`保存`、`制御`、`停止対策` のどれか
- その技術は `装置` の話か `ソフトウェア層` の話か
- 可用性を上げたいのか、性能を上げたいのか
- `どこが壊れても止まりにくいか` を具体的に言えるか
