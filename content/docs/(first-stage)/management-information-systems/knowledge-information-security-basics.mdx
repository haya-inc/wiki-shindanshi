## このページの役割

このページは、`情報セキュリティの基礎` を整理する知識ノードです。`何を守るか`、`どう守るか`、`どの脅威に効くか` を切り分けます。

## 学習のポイント

- まず `CIA` で守る対象を押さえる
- 次に `暗号化 / ハッシュ / 電子署名 / 認証` の役割を分ける
- そのうえで `マルウェア`、`ファイアウォール`、`VPN` を対策との対応で覚える

## 試験で何が問われるか

- `機密性`、`完全性`、`可用性` の違いを説明できるか
- `暗号化`、`ハッシュ`、`電子署名`、`PKI` の役割を言えるか
- `認証` と `認可` の違いを言えるか
- `マルウェア`、`ファイアウォール`、`VPN` が何を防ぐ仕組みかを説明できるか

## 定義と仕組み

### CIA

`機密性` は見られたくない相手に見せないこと、`完全性` は勝手に変えられないこと、`可用性` は必要なときに使えることです。まず `何を守る観点か` をここで固定します。

### 暗号化とハッシュ

`暗号化` は内容を読めない形に変えること、`ハッシュ` は元に戻せない形で要約値を作り、改ざん確認に使うことです。したがって、`隠す` と `同じか確かめる` は別の役割です。

### 電子署名と PKI

`電子署名` は、送信者本人かどうかと改ざんの有無を確認するための仕組みです。`PKI` は公開鍵基盤であり、電子証明書などを通じてそれを支えます。

### 認証とアクセス制御

`認証` は本人確認、`認可` は何をしてよいかの権限管理です。`パスワード`、`多要素認証`、`生体認証` は認証の手段として整理します。

### 脅威と対策

`マルウェア` は悪意あるプログラム、`ファイアウォール` は不正通信の遮断、`VPN` は安全な通信経路の確保です。脅威と対策を 1 対 1 で見ず、何を守るための仕組みかで読みます。

## 典型例

- 顧客情報を外部へ見せないのは `機密性`
- 改ざんされていないか確かめるのは `完全性`
- 証明書付きサイト接続は `電子署名 / PKI`
- 社外から社内へ安全に接続するのは `VPN`

## 典型的なつまずき

- `暗号化` と `ハッシュ` をどちらも隠す技術として覚える
- `電子署名` を暗号化そのものだと考える
- `認証` と `認可` の違いを曖昧にする
- `マルウェア` の種類と対策の目的を混ぜる

## 問題を解くときの観点

- 問われているのは `守る観点` か `具体的手段` か
- その技術は `見せない`、`改ざんを防ぐ / 見抜く`、`本人確認する` のどれか
- 脅威と対策が対応しているか
- 制度やガイドラインの話なら、基礎技術と実務指針を分けて読むか
