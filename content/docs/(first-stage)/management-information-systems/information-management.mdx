## このページの役割

このページは、`経営情報管理` という章全体のハブです。ここで章の全体像と学習順を確認し、詳細は `IT戦略`、`開発`、`運用`、`外部資源活用`、`意思決定支援` の知識ノードへ進む構成にします。

## 学習のポイント

- 最初の軸は `IT戦略・BPR・DX`、`開発手法`、`プロジェクト管理`
- 次に `運用管理・評価・監査` を、`開発後にどう回すか` の観点で整理する
- 後半で `外部資源活用` と `意思決定支援` を、`委託` と `データ活用` に分けて押さえる
- そのうえで `統計` と `更新論点` を補助線として使う

## この章で解けるようにしたいこと

1. `BPR` と `DX` を、目的と変化の大きさで区別できる
2. `ウォーターフォール`、`V字モデル`、`アジャイル`、`DevOps` を比較できる
3. `WBS`、`PERT`、`EVM` を、計画、日程、進捗管理の違いで使い分けられる
4. `ITIL`、`SLA`、`システム監査`、`内部統制` を、運用と評価の流れで説明できる
5. `アウトソーシング`、`SaaS`、`PaaS`、`IaaS`、`BI`、`DWH`、`OLAP` の役割を区別できる

## 2026-03-27 時点の最新確認メモ

- IPA の `共通フレーム2013` は公開年こそ古いですが、開発と運用を横断する共通枠組みとして、試験で今も基礎に置きやすい資料です。
- 経済産業省の `デジタルガバナンス・コード3.0` は `2024-09-19` 改訂で、DX 経営を `3つの視点・5つの柱` で整理しています。
- IPA の `情報システム・モデル取引・契約書` サイトは `2025-06-17` にサイト構成を見直し、第二版とアジャイル開発版へ整理されています。
- 総務省・経済産業省の `AI事業者ガイドライン` は `2025-03-28` 付 `第1.1版` が確認でき、AI 開発者、提供者、利用者それぞれの観点で読む構成になっています。
- IPA の `中小企業の情報セキュリティ対策ガイドライン` は `2026-03-27` 最終更新で `第4.0版` が公開されています。

## 典型的なつまずき

- DX を単なる IT 導入だと理解してしまう
- ウォーターフォール、アジャイル、DevOps を同じ粒度で雑に覚える
- 要件定義と設計、運用と監査の役割が混ざる
- クラウドの利用形態と契約・責任分界を分けて考えられない
- BI、DWH、OLAP の役割をまとめて覚えてしまう
