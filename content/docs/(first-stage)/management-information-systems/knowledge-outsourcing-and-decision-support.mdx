## このページの役割

このページは、`社外資源をどう使うか` と `データをどう意思決定へ変えるか` を整理する知識ノードです。`アウトソーシング`、`SaaS / PaaS / IaaS`、`モデル契約`、`BI`、`DWH`、`OLAP`、`データマイニング`、`IoT` を、委託と分析の2つの流れで切り分けます。

## 学習のポイント

- まず `内製` と `外部資源活用` を分ける
- 次に `SaaS / PaaS / IaaS` を、どこまで自社で持つかで整理する
- そのうえで `DWH`、`OLAP`、`BI`、`データマイニング` を、蓄積、分析、可視化で切る

## 試験で何が問われるか

- `アウトソーシング` と `内製` の違いを、責任と契約の観点で説明できるか
- `SaaS`、`PaaS`、`IaaS` を、利用者がどこまで管理するかで区別できるか
- `DWH`、`OLAP`、`BI`、`データマイニング` の役割を言えるか
- `IoT` を、データ収集と現場改善や意思決定支援の流れで読めるか

## 定義と仕組み

### 外部資源活用の考え方

外部資源活用では、`自社で作るか`、`委託するか`、`既存サービスを使うか` を選びます。試験では、単に費用の話ではなく、`責任分界`、`契約`、`柔軟性` をどう見るかが重要です。

### クラウドの利用形態

`SaaS` は完成したソフトウェアを利用する形、`PaaS` は開発や実行の基盤を利用する形、`IaaS` はサーバやネットワークなどのインフラを利用する形です。したがって、`どこまで自社が管理するか` を軸に切ると混ざりにくくなります。

### DWH、OLAP、BI、データマイニング

`DWH` は分析用にデータを集めて蓄積する仕組みです。`OLAP` は多面的に集計や分析を行う仕組み、`BI` は意思決定のために情報を見える化する仕組みです。`データマイニング` は、蓄積データから規則性や傾向を見つける分析手法です。

### IoT と意思決定支援

`IoT` は、機器やセンサから現場データを集める仕組みです。そのデータを `DWH` へ蓄積し、`OLAP` や `BI` で分析すると、保全、需要予測、在庫管理の判断へつなげやすくなります。

## 典型例

| 論点 | 典型的な見分け方 |
| --- | --- |
| `アウトソーシング` と `内製` | 自社で持つか、社外へ委ねるか |
| `SaaS / PaaS / IaaS` | どこまで自社が管理するか |
| `DWH` と `DB` | 日常処理用か、分析用蓄積か |
| `OLAP`、`BI`、`データマイニング` | 集計分析か、可視化か、規則発見か |

## 典型的なつまずき

- `SaaS`、`PaaS`、`IaaS` を全部 `クラウド` とだけ覚える
- `DWH`、`OLAP`、`BI` を全部 `分析ツール` として同じに見てしまう
- アウトソーシングを `丸投げ` とだけ理解し、契約や責任分界を見落とす
- `IoT` を機器の話だけで終わらせて、意思決定支援へつなげない

## 問題を解くときの観点

- 問われているのは `委託`、`利用形態`、`データ蓄積`、`分析`、`可視化` のどれか
- そのサービスは `完成品を使う` のか、`基盤を使う` のか、`インフラを使う` のか
- その仕組みは `集める`、`ためる`、`分析する`、`見せる` のどこを担うか
- 現場データが、どの判断につながるかまで言えるか
