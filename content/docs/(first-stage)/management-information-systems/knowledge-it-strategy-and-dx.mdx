## このページの役割

このページは、`経営課題と情報システムをどう結び付けるか` を整理する知識ノードです。`BPR`、`DX`、`IT戦略`、`ITガバナンス`、`ERP / SCM / CRM` を、目的と企業での使われ方で切り分けます。

## 学習のポイント

- まず `経営課題` と `情報システムの役割` を結び付ける
- 次に `BPR` と `DX` の違いを、変化の対象と広さで分ける
- そのうえで `IT戦略`、`ITガバナンス`、`主要システム` の位置づけを固定する

## 試験で何が問われるか

- `BPR` と `DX` を、何を変える考え方かで説明できるか
- `IT戦略` と `ITガバナンス` の違いを言えるか
- `ERP`、`SCM`、`CRM`、`SFA` を、どの業務をつなぐ仕組みかで区別できるか
- 情報システムを `効率化`、`見える化`、`競争力強化` のどこに使うかを読めるか

## 定義と仕組み

### 経営課題と情報システム

情報システムは、単なる機械化ではなく、`業務を速くする`、`情報を共有する`、`意思決定を支える` ために使います。試験では、まず `何の課題を解きたいのか` を見て、そのあとでどの仕組みが合うかを考えます。

### BPR と DX

`BPR` は業務プロセスそのものを抜本的に見直す考え方です。`DX` はデジタル技術を使って、業務だけでなく事業や顧客体験まで変える考え方です。したがって、`業務の再設計` が中心なら `BPR`、`事業変革` まで踏み込むなら `DX` と読むと崩れにくくなります。

### IT戦略と IT ガバナンス

`IT戦略` は、経営戦略を実現するために `どの情報システムへ投資し、何を優先するか` を決める考え方です。`ITガバナンス` は、その方針どおりに IT 投資や運用を統制する仕組みです。前者は `方向付け`、後者は `統制` と切るのが基本です。

### 主要な情報システム

| 仕組み | 何をつなぐか | 企業での使われ方 |
| --- | --- | --- |
| `ERP` | 会計、販売、購買、生産など基幹業務 | 全社の情報を一元化する |
| `SCM` | 調達、生産、在庫、物流 | 供給の流れを最適化する |
| `CRM` | 顧客情報、接点、購買履歴 | 顧客対応や継続取引を強める |
| `SFA` | 営業活動、案件進捗、行動履歴 | 営業活動を見える化する |

試験では、`誰のための仕組みか` と `何を良くするか` を先に切ると、略語だけで混ざりにくくなります。

## 典型例

| 論点 | 典型的な見分け方 |
| --- | --- |
| `BPR` と `DX` | 業務再設計が中心か、事業変革まで含むか |
| `IT戦略` と `ITガバナンス` | 方針を決める話か、統制して守る話か |
| `ERP` と `SCM` | 社内基幹業務の一元化か、供給連鎖の最適化か |
| `CRM` と `SFA` | 顧客関係の強化か、営業活動管理か |

## 典型的なつまずき

- `BPR` と `DX` をどちらも `IT化` とだけ覚える
- `IT戦略` と `ITガバナンス` を同じ `経営管理` として扱う
- `ERP`、`SCM`、`CRM`、`SFA` を全部 `業務システム` として雑にまとめる
- 情報システムを `経営課題の解決手段` ではなく `用語暗記` にしてしまう

## 問題を解くときの観点

- 問われているのは `方向付け`、`業務改革`、`統制`、`業務支援` のどれか
- その仕組みは `社内全体`、`供給の流れ`、`顧客接点`、`営業活動` のどこを支えるか
- `効率化` だけでなく、`共有`、`見える化`、`意思決定` のどこに効くか
- `DX` なら、業務改善で終わるのか、事業や顧客体験まで変えるのか
