## このページの役割

<Callout title="このページの役割">
    このページは、`品質をどう見て、どう改善するか` を整理する解説ページです。数値を見る道具と、原因や方針を整理する道具を分ける戻り先です。
</Callout>

<Callout title="このページの読み方">
  「不良品が増えたが、原因がわからない」──こんなとき、まず何をすべきでしょうか？ パレート図で「どの不良が最も多いか」を特定し、特性要因図で「なぜ起きるのか」を分析し、管理図で「工程が安定しているか」を監視する。このページでは、QC 7つ道具と新QC 7つ道具の使い分けを理解し、工程能力指数の計算方法を学びます。
</Callout>

## 学習のポイント

- `QC7つ道具` は数値や事実の把握に強い
- `新QC7つ道具` は整理、方針、要因関係の見える化に強い
- `工程能力指数` は頻出の計算問題

## 試験で何が問われるか

- QC7つ道具と新QC7つ道具の **使い分け** を説明できるか
- 工程能力指数（Cp, Cpk）を **計算** できるか
- 管理図の読み方と異常判定ができるか

## QC 7つ道具 ── 数値データで現場を見る

| 道具 | 何をするか | 使う場面 |
| --- | --- | --- |
| パレート図 | 不良項目を大きい順に並べる | 重点的に改善すべき項目の特定 |
| 特性要因図（魚骨図） | 結果と原因の関係を整理 | 不良原因の洗い出し |
| ヒストグラム | データの分布を見る | ばらつきの把握 |
| 散布図 | 2つのデータの関係を見る | 相関の有無の確認 |
| 管理図 | 時系列の変化を監視 | 工程の安定性の監視 |
| チェックシート | データを簡単に集める | 不良の発生記録 |
| 層別 | データを要因で分ける | 原因の切り分け |

### 管理図の判定ルール

管理図には UCL（上方管理限界線）、CL（中心線）、LCL（下方管理限界線）があり、以下の場合に異常と判断します。

- 点が管理限界線の **外** に出る
- 連続する点が中心線の **片側** に 7点以上並ぶ（連）
- 点が **一方向** に連続して増加または減少する（傾向）

## 新QC 7つ道具 ── 言語データで計画を立てる

| 道具 | 何をするか | 使う場面 |
| --- | --- | --- |
| 親和図法（KJ法） | 言語データをグルーピング | 混沌とした問題の整理 |
| 連関図法 | 原因と結果の複雑な関係を整理 | 多因子の因果関係の把握 |
| 系統図法 | 目的→手段を階層的に展開 | 施策の具体化 |
| マトリックス図法 | 2つの要素の対応を表で整理 | 要素間の関連性の評価 |
| アローダイアグラム法 | 作業の順序と日程を図示 | PERT/CPM（日程計画） |
| PDPC法 | 想定される展開を事前に描く | リスク対応の計画 |
| マトリックスデータ解析法 | 数値データを多変量で分析 | 唯一の数値的手法 |

## 工程能力指数（Cp, Cpk）

工程能力指数は、工程のばらつきが規格幅の中に収まっているかを数値化する指標です。

### Cp（工程が中心にある場合）

$$
Cp = \frac{USL - LSL}{6\sigma}
$$

`USL` = 上側規格限界、`LSL` = 下側規格限界、`σ` = 工程の標準偏差

### Cpk（工程が片寄っている場合）

$$
Cpk = \min\left(\frac{USL - \bar{X}}{3\sigma},\ \frac{\bar{X} - LSL}{3\sigma}\right)
$$

**常に Cpk ≦ Cp** が成り立ちます。

### 判定基準

| Cp の値 | 判定 |
| --- | --- |
| Cp ≧ 1.67 | 十分すぎる（管理の簡略化を検討） |
| 1.33 ≦ Cp < 1.67 | 十分 |
| 1.00 ≦ Cp < 1.33 | やや不足（改善が望ましい） |
| Cp < 1.00 | 不足（**工程改善が必要**） |

### 計算例

```
【データ】
  USL = 50mm、LSL = 40mm、工程平均 X̄ = 46mm、σ = 1.5mm

Cp = (50−40) ÷ (6×1.5) = 10 ÷ 9 ≒ 1.11（やや不足）

Cpk = min[(50−46)÷(3×1.5), (46−40)÷(3×1.5)]
    = min[4÷4.5, 6÷4.5]
    = min[0.89, 1.33]
    = 0.89（不足 → 工程改善が必要）

→ Cpは1.11で「やや不足」だが、工程が上寄りに片寄っているため
  Cpkは0.89でさらに悪い判定になる
→ 工程の中心を下げる（片寄りを修正する）改善が有効
```

## TQM（Total Quality Management）

TQM は、品質を全社的に作り込む経営管理の考え方です。QC サークルのような現場活動だけでなく、経営トップのリーダーシップ、全部門の参加、PDCA サイクルの継続的運用を含みます。

| 要素 | 内容 |
| --- | --- |
| 全員参加 | トップから現場まで全員が品質に責任を持つ |
| プロセス重視 | 結果だけでなくプロセスを管理する |
| PDCA | 継続的な改善サイクル |
| 顧客志向 | 顧客の要求を基準にする |

## 典型的なつまずき

- QC7つ道具と新QC7つ道具を一覧暗記だけで終える → 「数値 vs 言語」「分析 vs 計画」で分ける
- 管理図を単なるグラフの一種だと思ってしまう → 工程の **安定性** を監視する道具
- TQM を QC サークルと同じものとして扱う → TQM は **全社的経営活動**、QCサークルは現場の小集団活動
- Cp と Cpk の違いを忘れる → Cp は片寄り無視、Cpk は片寄り考慮
- Cp が 1.0 以上でも Cpk が 1.0 未満になるケースを見落とす → 片寄りがあると Cpk は低下する

## 問題を解くときの観点

- 問われているのは `数値の把握` か `原因整理` か `全社活動` か
- 工程の異常を見たいのか、改善テーマを整理したいのか
- Cp と Cpk のどちらを使うべきか（片寄りの有無）

## 確認問題

<Callout title="問1：工程能力指数の計算">
    USL = 100、LSL = 80、σ = 3。Cp を求め、判定せよ。

    **解答**：Cp = (100−80)÷(6×3) = 20÷18 ≒ 1.11。判定:「やや不足」（改善が望ましい）。
</Callout>

<Callout title="問2：Cpk の計算">
    USL = 100、LSL = 80、X̄ = 93、σ = 3。Cpk を求めよ。

    **解答**：Cpk = min[(100−93)÷(3×3), (93−80)÷(3×3)] = min[7/9, 13/9] = min[0.78, 1.44] = 0.78。工程が上寄りに片寄っており、改善が必要。
</Callout>

<Callout title="問3：QC道具の選択">
    「不良品の発生原因が複数考えられるが、どの原因が最も影響が大きいか特定したい」場合に使うべき道具はどれか。

    **解答**：パレート図。不良項目を発生頻度順に並べ、累積比率で重点項目を特定する。原因の洗い出し自体には特性要因図を併用する。
</Callout>
