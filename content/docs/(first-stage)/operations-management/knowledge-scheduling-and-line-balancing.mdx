## PERT/CPM ── 最短工期とクリティカルパス

PERT（Program Evaluation and Review Technique）/CPM（Critical Path Method）は、作業の前後関係をネットワーク図で表し、最短工期と余裕のない経路（クリティカルパス）を求める手法です。

### 基本用語

| 用語 | 意味 |
| --- | --- |
| 最早開始時刻（ES） | その作業を最も早く始められる時刻 |
| 最早完了時刻（EF） | ES + 作業時間 |
| 最遅完了時刻（LF） | その作業を遅くとも完了すべき時刻 |
| 最遅開始時刻（LS） | LF − 作業時間 |
| 総余裕時間（TF） | LS − ES（= LF − EF） |
| クリティカルパス | TF = 0 の作業を結ぶ最長経路 |

### 計算手順

```
【フォワードパス（前進計算）】── 最早時刻を求める
① 開始ノードの ES = 0
② 各作業の EF = ES + 作業時間
③ 合流ノードの ES = 流入する全作業の EF の最大値

【バックワードパス（後退計算）】── 最遅時刻を求める
④ 最終ノードの LF = 最短工期（フォワードパスの結果）
⑤ 各作業の LS = LF − 作業時間
⑥ 分岐ノードの LF = 流出する全作業の LS の最小値

【余裕時間】
⑦ TF = LS − ES（= LF − EF）
⑧ TF = 0 の作業がクリティカルパス上の作業
```

### 計算例

```
【ネットワーク】
  作業A(3日) → 作業C(4日) → 作業E(2日)
  作業B(5日) → 作業D(3日) → 作業E(2日)
  作業A(3日) → 作業D(3日)

【フォワードパス】
  A: ES=0, EF=3
  B: ES=0, EF=5
  C: ES=3, EF=7
  D: ES=max(3,5)=5, EF=8
  E: ES=max(7,8)=8, EF=10

→ 最短工期 = 10日

【バックワードパス】
  E: LF=10, LS=8
  D: LF=8, LS=5
  C: LF=8, LS=4
  A: LF=min(4,5)=4, LS=1
  B: LF=5, LS=0

【余裕時間（TF = LS − ES）】
  A: 1−0=1日  B: 0−0=0日  C: 4−3=1日  D: 5−5=0日  E: 8−8=0日

→ クリティカルパス: B → D → E（TF=0の経路）
→ この経路上の作業が遅れると全体工期が延びる
```

### 3点見積法

作業時間が不確実な場合、楽観値（a）、最頻値（m）、悲観値（b）から期待時間を算出します。

$$
t_e = \frac{a + 4m + b}{6}
$$

## ジョンソン法 ── 2機械フローショップの最適順序

ジョンソン法は、2 台の機械で n 個のジョブを処理する場合に、総所要時間を最小化する順序を求める方法です。

### アルゴリズム

```
① 全ジョブの機械1と機械2の処理時間をリストアップ
② 全体の最小値を探す
③ その値が機械1にあれば → そのジョブを先頭から順に入れる
④ その値が機械2にあれば → そのジョブを末尾から順に入れる
⑤ 決定したジョブを除外して②に戻る
⑥ 全ジョブの順序が決まるまで繰り返す
```

### 計算例

```
【データ】4つのジョブ
  ジョブ | 機械1 | 機械2
  J1     |   3   |   6
  J2     |   8   |   2
  J3     |   5   |   7
  J4     |   4   |   3

手順1: 全体の最小値 → J2の機械2=2
  → 機械2なので末尾に配置: [_, _, _, J2]

手順2: 残り{J1,J3,J4}の最小値 → J1の機械1=3
  → 機械1なので先頭に配置: [J1, _, _, J2]

手順3: 残り{J3,J4}の最小値 → J4の機械2=3
  → 機械2なので末尾から配置: [J1, _, J4, J2]

手順4: 残り{J3}
  → [J1, J3, J4, J2]

最適順序: J1 → J3 → J4 → J2
```

## ラインバランシング

ラインバランシングは、各工程への作業配分を最適化し、ライン全体の効率を高める手法です。

### タクトタイム

$$
\text{タクトタイム} = \frac{\text{稼働時間}}{\text{必要生産数}}
$$

タクトタイムは「市場が要求する1個あたりの生産ペース」です。各工程の作業時間はこのタクトタイム以内に収める必要があります。

### サイクルタイム

実際のラインにおけるサイクルタイム（CT）は、最も時間のかかる工程（ボトルネック工程）の作業時間です。

$$
CT = \max(\text{各工程の作業時間})
$$

### ライン編成効率

$$
\text{ライン編成効率} = \frac{\text{各工程の作業時間の合計}}{\text{工程数} \times CT} \times 100\%
$$

### バランスロス率

$$
\text{バランスロス率} = 100\% - \text{ライン編成効率}
$$

### 計算例

```
【データ】
  稼働時間 480分/日、必要生産数 120個/日
  工程A: 3分、工程B: 4分、工程C: 2分、工程D: 3分

手順1: タクトタイム
  = 480 ÷ 120 = 4分/個

手順2: サイクルタイム（ボトルネック）
  CT = max(3, 4, 2, 3) = 4分（工程B）

手順3: ライン編成効率
  = (3+4+2+3) ÷ (4×4) × 100
  = 12 ÷ 16 × 100
  = 75%

手順4: バランスロス率
  = 100 − 75 = 25%

→ 25%の時間がムダ待ちになっている
→ タクトタイム=CT=4分なので、生産ペースは市場要求ぴったり
→ 工程Bがボトルネック。改善するなら工程Bの作業時間短縮
```

### 最小工程数の理論値

$$
\text{最小工程数} = \lceil \frac{\text{作業時間の合計}}{CT} \rceil
$$

（切り上げ）

## ディスパッチングルール

ジョブの処理順序を決めるルールです。

| ルール | 基準 | 特徴 |
| --- | --- | --- |
| SPT（最短処理時間） | 処理時間が短い順 | 平均フロータイム最小 |
| EDD（最早納期） | 納期が早い順 | 納期遅れの最大値を抑制 |
| FCFS（先着順） | 到着順 | 公平だが効率は良くない |

## 典型的なつまずき

- PERT/CPM を計算手順だけで覚えて意味を失う → クリティカルパスは「**全体を遅らせている経路**」
- サイクルタイムとタクトタイムを混同する → タクトタイムは **市場要求**、サイクルタイムは **現場の実態**（ボトルネック）
- ボトルネック工程を見ずに平均で考えてしまう → ラインの速度は **最も遅い工程** で決まる
- ジョンソン法で「機械1なら先頭、機械2なら末尾」を逆にする → **小さい値が機械1→先頭、機械2→末尾**
- 余裕時間がマイナスになると慌てる → 計算ミスを疑う。TF は 0 以上

## 問題を解くときの観点

- 問われているのは `全体日程` か `順序` か `工程配分` か
- 全体を遅らせているのはどこか（クリティカルパス）
- 計画値と実績値を混ぜていないか
- タクトタイムと CT の関係はどうか（CT ≦ タクトタイムが理想）

## 確認問題

<Callout title="問1：クリティカルパス">
    作業A(2日)→C(5日)、作業B(4日)→D(3日)、作業C→E(1日)、作業D→E(1日)の場合、最短工期とクリティカルパスを求めよ。

    **解答**：A→C→E: 2+5+1=8日。B→D→E: 4+3+1=8日。どちらも8日で、両方がクリティカルパス。最短工期は8日。
</Callout>

<Callout title="問2：ライン編成効率">
    5工程のラインで、各工程の作業時間が 6分、8分、5分、7分、4分。ライン編成効率を求めよ。

    **解答**：CT = max(6,8,5,7,4) = 8分。合計 = 6+8+5+7+4 = 30分。編成効率 = 30÷(5×8)×100 = 30÷40×100 = 75%。
</Callout>

<Callout title="問3：ジョンソン法">
    3つのジョブ（J1: 機械1=2,機械2=5、J2: 機械1=6,機械2=3、J3: 機械1=4,機械2=4）の最適順序をジョンソン法で求めよ。

    **解答**：最小値=J1の機械1(2)→先頭: [J1,_,_]。次の最小値=J2の機械2(3)→末尾: [J1,_,J2]。残りJ3: [J1,J3,J2]。最適順序: J1→J3→J2。
</Callout>
