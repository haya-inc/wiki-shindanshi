# 簿記2級wiki キックオフ計画

## 前提

- 別リポジトリ、`/` をルートにする
- Fumadocs + Next.js 16 + pnpm
- 診断士wikiで確立した6ステップ工程を適用する

## Phase 0: リポジトリ初期化（1セッション）

1. `pnpm create fumadocs-app` でscaffold
2. `AGENTS.md` を配置
3. `docs/` に管理ファイル3点を配置
   - `wiki-progress-tracker.md`
   - `wiki-coverage-registry.md`
   - `wiki-review-log.md`
4. `content/docs/` にディレクトリ構造と各 `index.mdx` を作成
5. `pnpm lint` + `pnpm build` で動作確認
6. 初期コミット

## Phase 1: S群の特定と着手（1〜2セッション）

### S群の候補（出題頻度が最も高い論点）

商業簿記:
- **連結会計**（資本連結・のれん・内部取引消去・未実現利益） — 毎回大問1題分出題
- **有形固定資産**（取得・減価償却・売却・除却） — 仕訳問題の定番
- **引当金**（貸倒・修繕・賞与） — 仕訳問題の定番
- **税効果会計** — 精算表の常連

工業簿記:
- **標準原価計算**（差異分析） — 毎回大問に出題
- **総合原価計算**（平均法・先入先出法・仕損減損） — 毎回大問に出題
- **CVP分析**（損益分岐点・安全余裕率） — 頻出
- **製造間接費の配賦差額分析** — 頻出

### 作業順序

1. S群の知識ノードを5層フォーマットで作成（ステップ3）
2. 即座に教材型に書き直し（ステップ4）
3. サンプル問題との突合（ステップ5）
4. ナビ整備（ステップ6）

## Phase 2: A群の追記（2〜3セッション）

商業簿記:
- 有価証券（売買目的・満期保有・子会社関連・その他）
- リース取引
- 外貨建取引
- 収益認識（複数履行義務・変動対価）
- 株式会社会計（資本金・剰余金・配当）
- 本支店会計
- 決算整理（精算表・財務諸表作成）

工業簿記:
- 個別原価計算（仕損費含む）
- 部門費計算（直接配賦法・相互配賦法）
- 費目別計算（材料費・労務費・経費の予定計算）
- 直接原価計算（固定費調整）
- 工程別総合原価計算

## Phase 3: B群の追記 + 全体仕上げ（1〜2セッション）

- 残りの細目（商品売買の細目、手形、債務保証、未決算、営業費、製品受払い、工場会計独立）
- 出題ノード（大問形式別の解き方：仕訳問題、精算表、財務諸表作成、標準原価差異分析）
- 演習ノード（章ごとのオリジナル確認問題）
- 要点整理（比較表・チェックリスト）
- サンプル問題傾向分析ページ

## 診断士wikiとの関係

### 流用可能なコンテンツ

| 診断士wikiのノード | 簿記2級での対応 | 流用度 |
|-----------------|-------------|-------|
| 簿記の基礎（3ノード） | 商業簿記の基礎（3級範囲が主） | 低（3級範囲） |
| 企業会計（5ノード） | 商業簿記の諸取引・決算 | 中（概念は流用可、仕訳の深さが不足） |
| 原価計算（4ノード） | 工業簿記のほぼ全体 | 中（概念は流用可、計算の深さが不足） |

**注意**: 診断士wikiの財務・会計ノードは「4択で選択肢を絞る」レベルで書かれている。
簿記2級は「仕訳を書ける」レベルが必要なので、仕訳パターンと転記の層を大幅に追加する必要がある。
概念の説明とイメージ作りの部分は流用できるが、コピーではなく参照しながら簿記2級向けに書き直す。
