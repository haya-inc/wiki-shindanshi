# wiki 完全網羅ロードマップ

更新日: 2026-03-28

## 目的

この文書は、公開 wiki が `これさえ読めば全ての問題に応えられる` 状態にどこまで近づいているかを、現行の coverage 台帳を基準に判断するためのものです。

ここでいう `完全網羅` は、単に章ページがあることではありません。次の 3 条件を満たす状態を指します。

- `_book/_作成メモ/目次_確定版.md` の全細目に、公開側の戻り先がある
- 過去問や類題を見たときに、`どのページへ戻れば解けるか` をすぐ辿れる
- 更新論点は一次情報確認と確認日が整理され、安定論点と混ざらない

## 現状評価

結論として、現状は `範囲の穴と財務の大きな戻り先不足は解消し、第2次索引と事例Ⅰ〜Ⅲの実年度小問例も揃ったので、残件は事例別の悪文例 / 改善例と横断付録の逆流に集約された` 段階です。

2026-03-28 時点の coverage 件数は次です。

- `独立ノードあり`: 161
- `章ページ+演習あり`: 0
- `章ページで対応`: 3
- `章内に一部含む`: 0
- `未対応`: 0

この数字から言えることは明確です。

- `未対応 0` なので、範囲漏れは実質なくなった
- `章ページ+演習あり 0` なので、主要論点の戻り先不足はかなり解消した
- 残件は `第2次の年度別小問例と索引行の拡充`、`横断付録の逆引き`、`章ページ止まり 3 件` に集約された

## まだ足りないところ

### 1. 第2次試験は索引と小問例が揃い、次は悪文例と短文骨子の厚みが要る

`設問型`、`答案骨子`、`例題`、`誤答例`、`年度差` に加え、`年度 / 問題番号 / 設問型 / 必要知識 / 誤答パターン / 対応ページ` で逆引きする `設問索引` と、`事例Ⅰ〜Ⅲ` の実年度小問例は揃いました。

ただし、今は `戻る先が分かる` 段階です。`悪文例`、`改善例`、`短文骨子`、`事例Ⅳ` の行を足して、戻った先で `何を直すか` まで同じページ群で決められる余地があります。

### 2. 横断付録は入口として強いが、逆引きはまだ伸びしろがある

`重要用語集` と `重要公式集` は初版があり、科目別の導線も存在します。ただし、略語、類義語、誤答逆引き、章ハブからの往復はまだ強化余地があります。

ここは `独立ノードが揃ってから厚くする` 方が手戻りが少ないため、現段階で着手しやすい残件です。

### 3. 章ページ止まりの残りは 3 件だけ

残る `章ページで対応` 3 件は、次の `概要系ページ` に限られます。

- `付録C 試験概要と受験ガイド`
- `第22章 2次試験の全体像`
- `第22章 時間配分と解答戦略`

主要論点の不足ではないため優先度は下がりますが、完全網羅の最終盤ではここも `索引` または `補助ページ` で補う余地があります。

## 優先順位

今の優先順位は次です。

1. `重要用語集` と `重要公式集` から `事例Ⅳ`、1次知識ノード、索引へ戻る導線を増やす
2. `事例Ⅰ〜Ⅳ` の悪文例、改善例、短文骨子と索引行を増やし、`設問索引` を事例Ⅳと次年度まで広げる
3. `受験ガイド` と `第2次試験 overview` の残る章ページ止まりを、必要なら索引または個別ノードで補う

この順番を採る理由は、`第2次の逆引き` と `実年度小問例` の初版ができており、次の大きなボトルネックが `横断付録から本文へ戻る精度` と `戻った先で何を直すかの具体例不足` に移っているからです。`独立ノードが揃った今が横断付録を伸ばす最も効率のよい段階` でもあります。

## 実行方針

### フェーズ1: 第2次試験の悪文例、短文骨子、索引行

最初に対象にするのは次です。

- `事例Ⅰ`
- `事例Ⅱ`
- `事例Ⅲ`
- `事例Ⅳ`

持つべき列は次の 6 つです。

- `年度`
- `問題番号`
- `設問型`
- `必要知識`
- `誤答パターン`
- `対応ページ`

ここでは `ページ追加` より、`既にある設問索引` と `設問型 / 答案骨子` を `悪文例 / 改善例 / 短文骨子` でつなぎ、索引の行を増やすことが重要です。

### フェーズ2: 横断付録の逆引き強化

対象は次です。

- `重要用語集`
- `重要公式集`
- `過去問テーマ索引`

ここでは、`略語`、`類義語`、`誤答逆引き`、`章ハブからの戻り先` を増やします。用語や式を見たときに、`どの知識ノードへ戻るべきか` が一段早く決まる状態を目指します。

### フェーズ3: 残る章ページ止まり 3 件の補助

ここはページ数を増やすこと自体が目的ではありません。`索引で十分か`、`短い補助ページが必要か` を見て、最小差分で補います。

## 実行原則

- `1バッチ1テーマ` で進める
- 追加ページより `戻り先が明確になったか` を優先する
- `ページ数` ではなく、coverage 台帳と索引の逆引きしやすさで前進を測る
- 更新論点は `章ハブまたは科目ハブに日付を寄せる`
- 追加時は `meta.json`、索引、進捗台帳、coverage 台帳、レビュー記録、必要なら UI テストまで同時に更新する

## 完了判定

`完全網羅にかなり近い` と言える目安は次です。

- `章ページで対応` が 10 未満
- `章ページ+演習あり` が 5 未満
- `第2次試験` に設問索引があり、主要設問に戻り先がある
- `重要用語集` と `重要公式集` から主要論点へ逆引きできる

今の段階では、`章ページで対応 10 未満`、`章ページ+演習あり 5 未満`、`第2次試験に設問索引がある` はすでに満たせています。ここからは、`横断付録の逆引き` と `第2次の悪文例 / 改善例 / 索引行の拡充` に作業を集中させるのが最も効率的です。
